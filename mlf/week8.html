Lecture 1: Pillars of Machine Learning

Foundational Pillars of Machine Learning

Machine Learning is built on 3 core mathematical pillars:

Linear Algebra + Probability + Optimization

These work together to:

  • Understand data
  • Handle uncertainty
  • Make best decisions

Linear Algebra – Understanding Data Structure

Purpose: Helps us understand relationships between variables (features).

Key Ideas

  • Data representation:
    • Vectors → single data point
    • Matrices → dataset
  • Linear Relationship: one variable changes proportionally with another
    • Example: height ↑ → weight ↑
y = mx + c
In ML form: y = wTx + b
Where: x = input features, w = weights, b = bias

Why Important?

Forms the base of models like:

  • Linear Regression
  • Neural Networks

Probability – Handling Uncertainty

Purpose: Models uncertainty (noise) in real-world data.

Why Noise Exists

  • Measurement errors
  • Missing values
  • Natural variations

Key Concepts

  • Random Variables
  • Probability Distribution
  • Expectation (Mean)
Probability: P(A) = favorable outcomes / total outcomes

Gaussian (Normal Distribution): f(x) = (1 / √(2πσ²)) · e−(x−μ)² / (2σ²)
Where: μ = mean, σ = standard deviation

Why Important?

Helps models:

  • Handle imperfect data
  • Predict realistically
  • Avoid overconfidence

Optimization – Finding the Best Model

Purpose: Finds the best possible model from many choices.

Problem: There are infinite possible models (lines/curves). Which one is best? → Optimization solves this.

Key Concepts

  • Loss Function (Error measure)
  • Gradient Descent (learning method)
Mean Squared Error (MSE): J(w) = (1/n) ∑ (yi − ŷi)²

Gradient Descent Update: w = w − α · ∂J/∂w
Where: J(w) = loss, α = learning rate

Why Important?

  • Chooses best parameters
  • Improves model accuracy
  • Core of training ML models

Putting It All Together (Example)

Height → Weight Prediction

Step 1: Linear Algebra — Model relationship: Weight = w·Height + b

Step 2: Probability — Accounts for: same height → different weights; noise in real data

Step 3: Optimization — Finds best w and b; minimizes prediction error

Linear Algebra → "What is the pattern?"
Probability → "Why isn't it perfect?"
Optimization → "What is the best answer?"

Lecture 2: Introduction to Optimization

Optimization is the heart of Machine Learning. It helps convert data → decisions.

What is Optimization?

Definition: Optimization is the process of finding the best possible solution from many choices.

In Machine Learning:

  • Find best model
  • Find best parameters (weights)
  • Find best predictions

"Best" means: Minimize loss (error) OR Maximize reward (accuracy, profit, etc.)

Why Optimization is Important?

  • Every ML algorithm depends on it
  • Works for different tasks: Classification, Regression, Clustering
  • Provides a common mathematical framework
Cow and Grass Example (Intuition)
Setup: Cow position: (20, 30), Grass position: (40, 40)
Goal: Find position (x1, x2) where cow gets closest to grass.

Objective Function (Distance): f(x) = (x1−40)² + (x2−40)² — Minimize this distance.

Constraints:
1. Rope Constraint (Circle Region): (x1−20)² + (x2−30)² ≤ 100
2. Fence Constraint (Boundary Line): x1 ≤ 25
Cow must stay inside allowed region.

General Optimization Framework

Every optimization problem has 3 main components:

  1. Objective Function — Function to optimize (minimize/maximize). Examples: Distance, Loss function (MSE, Cross-Entropy)
  2. Variables (Parameters) — Values we are trying to find: x ∈ RD, x = (x1, x2, ..., xD)
  3. Constraints — Restrictions on values of x
    • Inequality Constraints: gi(x) ≤ 0 — Example: (x1−20)² + (x2−30)² − 100 ≤ 0
    • Equality Constraints: hj(x) = 0 — Example: Budget fixed, exact resource usage

Standard Form

General Optimization Problem: Minimize f(x)
Subject to: gi(x) ≤ 0 (i = 1,2,...,k), hj(x) = 0 (j = 1,2,...,L)

Important Conversions

  1. Maximization → Minimization: Maximize f(x) = Minimize −f(x)
  2. Constraint Conversion: Any constraint can be written as g(x) ≤ 0, h(x) = 0

In Machine Learning Context — Example: Linear Regression

  • Objective: min ∑(y − ŷ)²
  • Variables: Weights w, bias b
  • Optimization finds: Best w, b that minimize error

ComponentMeaningExample
Objective FunctionWhat to optimizeDistance, Loss
VariablesUnknownsx, w, b
ConstraintsRestrictionsRope, Fence
Final Summary: Optimization is like:
Target → Objective function | Rules → Constraints | Search → Finding best solution

One-Line Summary: Optimization = Finding the best solution under given conditions

Lecture 3: Solving an Unconstrained Optimization Problem (Part 1)

What is an Optimization Problem?

It consists of:

  • Objective Function → what to minimize/maximize
  • Constraints → restrictions (optional)

Unconstrained Optimization

  • No restrictions on variables
  • Only focus: minimize or maximize f(x)

Traditional Method (School Calculus)

Steps:

  1. Find derivative
  2. Set it to zero
  3. Solve for x: f′(x) = 0

Works well for simple functions. Limitations: Hard for complex equations, not scalable for ML problems.

Why Iterative Methods?

  • Computers cannot "think" like humans
  • They follow step-by-step updates

Iterative Optimization Idea: Start with an initial guess, improve it step-by-step, move toward minimum.

Direction Concept

What is Direction d? Tells which way to move — Left (negative), Right (positive)

Best Direction: d = −f′(x) — Points toward minimum

Update Rules (Important Formulae)

General Update Rule: xt+1 = xt + d
Derivative-Based Rule: xt+1 = xt − f′(xt)

Derivatives (Examples)

  • Example 1: f(x) = (x−5)², f′(x) = 2(x−5)
  • Example 2: f(x) = 3x⁶ + 2x⁵ + 3x³ + 5x² + 2, f′(x) = 18x⁵ + 10x⁴ + 9x² + 10x
Example 1: Simple Analytical Solution — f(x) = (x−5)²
Minimum = 0 at x = 5. Calculus: 2(x−5) = 0 ⇒ x = 5

Example 2: Complex Polynomial — f(x) = 3x⁶ + 2x⁵ + 3x³ + 5x² + 2
Derivative equation: 18x⁵ + 10x⁴ + 9x² + 10x = 0 — Very hard to solve manually → Need iterative methods

Example 3: Iterative Failure (Oscillation) — f(x) = (x−5)², x0 = 10
Iteration 1: f′(10) = 10 ⇒ x1 = 0
Iteration 2: f′(0) = −10 ⇒ x2 = 10
Keeps jumping: 10 → 0 → 10 → 0. Problem: Step size too large

Key Problem

  • Direction is correct
  • Step size too large

Quick Revision Table

ConceptKey Idea
UnconstrainedNo restrictions
Directiond = −f′(x)
Update Rulext+1 = xt − f′(xt)
ProblemOvershooting
SolutionControl step size
Final Conclusion: Optimization depends on correct direction (− derivative) and proper step size.
One-Line Summary: Wrong step size can break optimization even if direction is correct.

Lecture 4: Solving an Unconstrained Optimization Problem (Part 2)

Optimization Step Size (η)

In optimization:

  • Direction −f′(xt) → tells where to go
  • Step size ηt → tells how far to go

Both are equally important!

Why Step Size Matters?

1. Overshooting (Oscillation Problem)

If step size is too large: algorithm jumps over minimum, keeps oscillating.

2. Too Small Step Size (Slow / Never Reach)

If step size decreases too fast: moves very slowly, may never reach minimum.

Goal of Step Size

A good step size must:

  • Decrease over time (avoid oscillation)
  • Not decrease too fast (ensure reachability)

Main Formula

General Update Rule: xt+1 = xt − ηt f′(xt)
Where: ηt = step size (learning rate), f′(xt) = derivative

Step Size Strategies

1. Geometric Step Size (Failure Case)

ηt = 1/2t   Sequence: 1, 1/2, 1/4, 1/8,...
Property: ∑ηt = 2 (Total movement is limited / finite)
Start: x0 = 2, Target: x* = 5, Direction: +1
Steps: 2 → 3 → 3.5 → 3.75 ... Final result → approaches 4 only. Cannot reach 5.

2. Harmonic Step Size (Successful Case)

ηt = 1/(t+1)   Sequence: 1, 1/2, 1/3, 1/4,...
Property: ∑ηt = ∞ (Total movement is unlimited)

Result: Step sizes decrease slowly. Algorithm can reach minimum and avoid oscillation.

Key Concept: Cumulative Step Sum

Step TypeSumResult
GeometricFinite✗ Cannot reach minimum
HarmonicInfinite✓ Can reach minimum

Conflicting Objectives

GoalReason
Decrease over timeAvoid oscillation
Not too fast decreaseEnsure reachability
Final Intuition: Think of optimization like walking to a destination:
Big steps → overshoot | Tiny steps → never reach | Balanced steps → reach smoothly

One-Line Summary: Good step size = small enough to stabilize, big enough to reach

Lecture 5: Basic Algorithm for Unconstrained Optimization — Gradient Descent

Definition: Gradient Descent is a first-order iterative optimization algorithm used to minimize a function.

  • First-order → uses only derivative (slope)
  • Works step-by-step to reach minimum value

Intuition: You are on a hill. You move in the steepest downward direction. Eventually reach a valley (minimum).

How It Works

Update Rule (Most Important): xt+1 = xt − ηt f′(xt)
Where: xt → current value, f′(xt) → slope (derivative), ηt → step size (learning rate)

Step Size (Learning Rate)

Recommended: ηt = 1/(t+1) — Prevents oscillation, ensures convergence.

Multidimensional Case

For higher dimensions: Derivative → Gradient (∇f)

Update Rule: xt+1 = xt − ηt ∇f(xt)
Gradient = vector of partial derivatives

Convergence

As iterations increase: algorithm moves closer to minimum; step size decreases → stabilizes.

Local vs Global Minimum

  • Global Minimum: f(x*) ≤ f(x) ∀ x — Absolute best solution
  • Local Minimum: f(x̂) ≤ f(x) for nearby points — Best only in a small region

Formal Definition: There exists ε > 0 such that f(x̂) ≤ f(x), ∀ x ∈ (x̂ − ε, x̂ + ε)

Key Limitation

Gradient Descent usually finds local minimum — not guaranteed global minimum.

Special Case: Convex Functions

Definition: A function with only one minimum. Local = Global. Result: Gradient Descent guaranteed to find best solution.

Example 1: Simple Quadratic Function f(x) = (x−5)² — Convex function. Result: Minimum at x = 5. Gradient Descent always finds it.

Example 2: Wiggly Function — Multiple peaks & valleys. Different starting points → different results. May get stuck in local minimum.

Hill Analogy: Sliding down a hill → stop in a valley. But a deeper valley may exist far away → local minimum problem.

Important Insight

Finding global minimum is very hard for general functions. In ML: local minimum is usually good enough.

Lecture 6: Gradient Descent and Taylor Series

Optimization (Unconstrained)

Goal: Minimize a function: min f(x) — No constraints on x, only focus: find lowest value.

Gradient Descent Algorithm

Idea: Move step-by-step in a direction that reduces function value.

Update Rule (Most Important): xt+1 = xt + ηt(−f′(xt)) = xt − ηt f′(xt)
Where: xt → current point, ηt → step size, −f′(xt) → descent direction

Step Size (η)

Small positive value (e.g., 0.1, 0.01)

Role: Controls how far to move. Must be not too large (oscillation), not too small (slow).

Descent Direction

Definition: A direction d is descent if: f(x + ηd) < f(x) — means function value decreases.

Taylor Series (Core Concept)

Definition: Expands function near a point:

f(x + ηd) = f(x) + ηd f′(x) + (η²d²/2) f″(x) + ...

First-Order Approximation (for small η): f(x+ηd) ≈ f(x) + ηd f′(x)
Change in Function Value: f(x+ηd) − f(x) ≈ ηd f′(x)
Condition for Descent: f(x+ηd) − f(x) < 0 ⇒ ηd f′(x) < 0
Since η > 0: d · f′(x) < 0

Why Choose Negative Derivative?

  • f′(x) can be positive or negative
  • Fixed direction won't work
Solution: d = −f′(x)
Proof: d · f′(x) = (−f′(x)) · f′(x) = −(f′(x))² ≤ 0
Always negative or zero

Guarantees: function decreases, correct direction always chosen.

Example: f(x) = (x−5)², Derivative: f′(x) = 2(x−5)
One Step Update: Suppose x = 8, η = 0.1: xnew = 8 − 0.1·2(8−5) = 8 − 0.6 = 7.4 — Moved closer to 5

Key Intuition

Gradient Descent works because Taylor series tells us how function behaves locally, and negative derivative gives best decreasing direction.

Lecture 7: Gradient Descent for Multivariate Functions

Generalization to Higher Dimensions

  • Earlier: Functions of a single variable f(x)
  • Now: Functions of multiple variables f(x1, x2, ..., xn)
  • In higher dimensions: Derivative → replaced by Gradient (∇f). We move in multi-dimensional space instead of a line.

Gradient (∇f)

The gradient is a vector of partial derivatives:

∇f = (∂f/∂x1, ∂f/∂x2, ..., ∂f/∂xn)

Key Points

  • Shows rate of change in all directions
  • Direction → steepest increase
  • Magnitude → how fast the function changes
  • For minimization, we use the negative gradient

Partial Derivatives

  • Derivative with respect to one variable at a time
  • Treat other variables as constants
For f(x1, x2) at point (a,b):
∂f/∂x1: derivative w.r.t. x1 (keep x2 constant)
∂f/∂x2: derivative w.r.t. x2 (keep x1 constant)

Gradient at (a,b): ∇f(a,b) = (∂f/∂x1, ∂f/∂x2)

Gradient Descent Update Rule

xt+1 = xt − η∇f(xt)

Components

  • xt: current position (vector)
  • xt+1: next position
  • η: learning rate (step size)
  • ∇f(xt): gradient at current point

We move in the direction of −∇f(xt) (steepest decrease).

Algorithm Behavior & Convergence

  • Always move downhill
  • Eventually reaches a local minimum
  • Works for unconstrained optimization problems

Important: May not find global minimum. Depends on starting point and learning rate (η).

Illustrative Example (Cow & Grass)
Function: d(x1, x2) = (x1−40)² + (x2−40)²
Minimum occurs at (40, 40). Gradient: ∇d = (2(x1−40), 2(x2−40))

Algorithm Steps:
  1. Start from any point
  2. Compute gradient
  3. Move in −∇d direction
  4. Repeat until reaching (40, 40)
Quick Revision Summary:
Gradient = multi-variable derivative | Direction of ∇f = steepest increase | Use −∇f for minimization
Update rule: xt+1 = xt − η∇f(xt) | Converges to a local minimum

Lecture 8: Taylor Series in Higher Dimensions

Idea (Connection with 1D)

  • In 1D, Taylor expansion uses the derivative f′(x)
  • In higher dimensions, variables become vectors

Replacement: x → vector x, d → direction vector, f′(x) → ∇f(x)

Taylor Series Approximation (First Order)

f(x + ηd) ≈ f(x) + η(dT∇f(x))

Meaning: Predicts value of function at a nearby point using current value f(x), direction d, and step size η.

Key Components

  • x, d: vectors in n-dimensional space
  • ∇f(x): gradient (vector of partial derivatives)
  • dT∇f(x): directional derivative (scalar)

Interpretation: Tells how fast the function changes in direction d.

Directional Derivative

dT∇f(x)

Interpretation: >0: function increases | <0: function decreases | =0: no change

Choosing a Descent Direction

Goal: Minimize function. Condition: dT∇f(x) < 0 — Ensures f(x+ηd) < f(x)

Gradient Descent Choice

Choice: d = −∇f(x)
Result: dT∇f(x) = −‖∇f(x)‖²

Key Insight: Norm squared is always positive. Result is always negative. Guaranteed decrease in function value (for small η).

Geometry of Directions

  • Gradient divides space into two halves:
    • dT∇f(x) > 0 → increase
    • dT∇f(x) < 0 → decrease
  • Infinite directions exist where function decreases (the "green region")

Steepest Descent

Among all possible directions, the best choice is: d = −∇f(x) — gives maximum decrease per step. Hence called the Steepest Descent Algorithm.

Unconstrained vs Constrained Optimization

Unconstrained Optimization: No restrictions on x. Best direction = −∇f(x).

Constrained Optimization: Must satisfy constraint g(x) ≤ 0.

Example (Cow with Rope): Cow can only move within a fixed radius. Even if −∇f(x) is best, it may break the constraint or move outside the allowed region. Need alternative directions → constraint-aware optimization methods.
Quick Revision Summary:
Taylor expansion in higher dimensions uses gradient instead of derivative.
Key formula: f(x+ηd) ≈ f(x) + η(dT∇f(x))
For minimization: choose direction where dot product < 0.
Best direction: −∇f(x) (steepest descent).
Constraints may require different strategies.
