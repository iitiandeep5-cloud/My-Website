Lecture 1: Bivariate and Multivariate Normal

Standard Normal Vector

A standard normal vector is a d-dimensional vector:

z = (z1, z2, ..., zd) where each component zi ~ N(0, 1) (independent).

Key Idea: All components are independent and follow the standard normal distribution.

Joint Density: f(z) = 1/(2π)d/2 exp(−½‖z‖²)
Where ‖z‖² = z1² + z2² + ... + zd².

Bivariate Normal Distribution (2D Case)

Describes two random variables x1, x2.

Construction (from standard normals):

x1 = z1,   x2 = ρz1 + √(1−ρ²) z2

Intuition: ρ controls correlation.

  • If ρ = 0: independent
  • If ρ = 1: perfectly linear
Matrix Form: X = AZ   ⇒   (x1, x2)T = [1   0 ; ρ   √(1−ρ²)] (z1, z2)T

Properties

  • Mean: E[x1] = 0, E[x2] = 0
  • Variance: Var(x1) = 1, Var(x2) = 1
  • Covariance: Cov(x1, x2) = ρ
Covariance Matrix: Σ = [1   ρ ; ρ   1]

Marginal & Conditional

  • Marginal: x1 ~ N(0,1)
  • Conditional: x2 | x1 ~ N(ρx1, 1−ρ²)

Interpretation: If x1 is fixed, the best estimate of x2 is ρx1.

General Multivariate Normal (MVN)

Definition: X = AZ + μ, where A is a transformation matrix, μ is a mean vector, and Σ = AAT is the covariance matrix.

Density Function: fX(x) = 1/((2π)d/2|Σ|1/2) exp(−½(x−μ)TΣ−1(x−μ))

Notation: X ~ N(μ, Σ)

Conditions on Covariance Matrix Σ

  • Symmetric: Σ = ΣT
  • Positive definite
  • Variances > 0
  • Covariance bounded: |Cov(x1,x2)| ≤ σ1σ2

Important Properties

Linear Transformations

  • Scalar: Y = aTX ~ N(aTμ, aTΣa)
  • Vector: Y = BX ~ N(Bμ, BΣBT)

Independence vs Correlation

  • In general: zero covariance ⇏ independence
  • In MVN: zero covariance ⇒ independence
Example: Standardization
Given covariance: Σ = [a²   ρab ; ρab   b²]
Scaling matrix: B = [1/a   0 ; 0   1/b]

This transforms data to unit variance while preserving correlation ρ.

Visualization

Shape of Distribution: Contour plots are ellipses centered at mean μ. Data points cluster along: x2 = ρx1. Stronger ρ → tighter ellipse.

Final Summary
Standard Normal: independent N(0,1) components
Bivariate Normal: correlation controlled by ρ
MVN: X ~ N(μ, Σ)
Covariance matrix: symmetric + positive definite
Linear transforms preserve normality
In MVN: zero covariance ⇒ independence
Shape: elliptical contours

Lecture 2: Estimation of Parameters Using ML

Introduction to Parameter Estimation

Setting: We assume a family of distributions: P = {pθ: θ ∈ Θ} where θ is the unknown parameter and Θ is the parameter space.

Data: Observations: x1, x2, ..., xn with the assumption: xi ~ pθ i.i.d.

Goal: Estimate the true parameter θ that generated the data. Exact value is uncertain → find the most likely parameter.

Maximum Likelihood Estimation (MLE)

Likelihood Function: Measures how likely data is for a given θ: L(θ) = P(x1,...,xn | θ)

For i.i.d. data: L(θ) = ∏i=1n pθ(xi)

Log-Likelihood: log L(θ) = ∑i=1n log pθ(xi)

Negative Log-Likelihood (NLL): r(θ) = −∑i=1n log pθ(xi)

Key Idea: Maximize likelihood ⇔ Maximize log-likelihood ⇔ Minimize NLL

MLE Examples

A. Bernoulli Distribution (Coin Toss)
Setup: xi ∈ {0,1}, θ = P(head)
PMF: pθ(x) = θx(1−θ)1−x

NLL Function: r(θ) = a log(1/θ) + (n−a) log(1/(1−θ)) where a = ∑xi (number of heads)
MLE Result: θ̂ML = a/n
Interpretation: fraction of heads.
B. Uniform Distribution
Setup: Data lies in interval [a, b], parameter θ = (a,b)
Density: pθ(x) = 1/(b−a)
Logic: Make interval as tight as possible to include all data points.

MLE Result: â = min(xi), b̂ = max(xi)
C. Normal Distribution (σ² = 1, μ unknown)
Density: f(x) = (1/√(2π)) e−½(x−μ)²
NLL (simplified): r(μ) = ∑i=1n ½(xi−μ)²

MLE Result: μ̂ = (1/n) ∑ xi
Interpretation: sample mean.
D. Normal Distribution (μ and σ² unknown)
Density: p(x) = (1/√(2πσ²)) e−(x−μ)²/(2σ²)
NLL: r(μ,σ²) = (n/2) log(σ²) + ½ ∑i=1n (xi−μ)²/σ²

MLE Results: μ̂ = (1/n) ∑ xi     σ̂² = (1/n) ∑(xi−μ)²

Note: This variance uses 1/n (MLE), not 1/(n−1).

Key Takeaways

Core Ideas

  • MLE finds parameter that best explains observed data.
  • Works by maximizing likelihood (or minimizing NLL).
  • Foundation of many machine learning algorithms.

Important Insights

  • Log simplifies calculations.
  • i.i.d. assumption is crucial.
  • Solutions often give intuitive results:
    • Bernoulli → fraction
    • Normal → mean & variance

Big Picture: Used in regression, classification, Gaussian Mixture Models (GMM). Backbone of modern machine learning.

Revision
Likelihood: product of probabilities | Log-likelihood: sum of logs | NLL: minimize for optimization
Bernoulli: θ̂ = a/n | Uniform: min & max of data
Normal: mean = average, variance = average squared deviation
MLE = "parameter that best fits data"

Lecture 3: Gaussian Mixture Models and Expectation Maximisation

Core Concepts

Definition: A Gaussian Mixture Model (GMM) is a combination of K Gaussian distributions, used when data has multiple clusters (multi-modal). Instead of modeling a single peak, GMM models multiple peaks.

Motivation

  • Single distributions (Normal, Exponential) → only one peak.
  • Real data often has multiple groups.
Example: Exam scores
Weak students → low scores | Average students → mid scores | Strong students → high scores
Result: multiple peaks.

Latent Variable (Hidden Variable): Introduce z = cluster ID. We observe x (data), but not z (which cluster generated x). Hence, GMM is a latent variable model.

Unidentifiability: Swapping labels of clusters does not change the distribution. Example: Cluster 1 ↔ Cluster 2 → same distribution. Labels are arbitrary.

Mathematical Formulation

GMM Density Function: fX(x) = ∑k=1K πk N(x; μk, Σk)

Parameters

  • K: number of clusters
  • πk: mixture weights, with ∑πk = 1
  • μk: mean of cluster k
  • Σk: covariance matrix
Multivariate Gaussian: N(x;μk,Σk) = 1/((2π)d/2|Σk|1/2) exp(−½(x−μk)TΣk−1(x−μk))

Negative Log-Likelihood (NLL): ∑n=1N −log(∑k=1K πkN(xn;μk,Σk))

Challenge: Hard to optimize because of the log-of-sum structure.

Expectation-Maximization (EM) Algorithm

Problem: To find parameters → need cluster labels. To find labels → need parameters. This is the "chicken-and-egg" problem.

Step 0: Initialization: Start with random πk, μk, Σk.

E-Step (Expectation): Compute responsibility γnk:

γnk = πkN(xn;μk,Σk) / ∑j=1K πjN(xn;μj,Σj)

Interpretation: Probability that point xn belongs to cluster k.

M-Step (Maximization):

Effective cluster size: Nk = ∑n=1N γnk
Updated Mean: μk = (1/Nk) ∑n=1N γnkxn
Updated Covariance: Σk = (1/Nk) ∑n=1N γnk(xn−μk)(xn−μk)T
Updated Weights: πk = Nk / N

Iteration: Repeat E-step → M-step until convergence.

Examples

1. Exam Scores Example

Student TypeMeanVarianceWeight
Unprepared−40.50.4
Random010.3
Prepared510.3

Observed: Histogram shows 3 peaks, but cluster membership is unknown. GMM learns all 3 clusters automatically.

2. 2D Clustering

  • Data forms two clouds (clusters).
  • Start: random circles.
  • After EM: means shift, covariances stretch to fit data.
  • Final result: clusters match actual structure.

Applications

  1. Clustering: Groups data into multiple clusters. More flexible than K-means (can model elliptical clusters).
  2. High-Dimensional Data: Works in 10D, 100D. Visualization impossible, but EM handles it mathematically.
Final Summary
GMM = mixture of K Gaussians. Models multi-modal data. Uses latent variable z.
Density = weighted sum of Gaussians. MLE is hard → solved using EM algorithm.

EM Algorithm: E-step → find probabilities (γ). M-step → update parameters. Repeat until convergence.

Key Insights: Soft clustering (probabilities, not hard labels). Handles complex data shapes. Foundation for many ML models.

Lecture 4: Laws of Large Numbers — Markov, Chebyshev, Hoeffding, Central Limit

Tail Bounds (Deviation from the Mean)

Tail bounds estimate the probability that a random variable deviates significantly from its mean.

1. Markov Inequality (Basic & General)

Condition: Works only for non-negative random variables X ≥ 0.

P(X ≥ t) ≤ E[X] / t

Key Idea: Uses only the mean. Very general but often loose.

Example: If E[X] = 10, then P(X ≥ 50) ≤ 10/50 = 0.2 — at most 20%.

Tightness Example: If P(X=50) = 1/5, P(X=0) = 4/5, then E[X] = 10. Bound = actual probability → tight.

2. Chebyshev Inequality (Uses Variance)

Condition: Works for any random variable, requires mean and variance.

P(|X−μ| ≥ t) ≤ σ²/t²

Key Idea: Measures deviation from mean. Probability decreases as ~1/t².

Example: If σ² = 4, t = 4: P(|X−μ| ≥ 4) ≤ 4/16 = 0.25

3. Hoeffding's Inequality (Very Strong)

Conditions: X1, ..., Xn are i.i.d. and bounded: a ≤ Xi ≤ b.

Formula (for sample mean): P(|X̄n − μ| ≥ ε) ≤ 2 exp(−2nε² / (b−a)²)

Key Insight: Probability decreases exponentially (~e−n). Much tighter than Chebyshev (~1/n).

Notions of Convergence

1. Convergence in Probability

Definition: limn→∞ P(|Xn − X| ≥ ε) = 0

Meaning: Probability of large deviation goes to 0. Eventually, values stay close to the limit.

2. Convergence in Distribution

Idea: The distribution (CDF) of Xn approaches that of X.

Key Insight: Weaker than convergence in probability. Focuses on distributional shape, not exact values.

Limit Laws

1. Weak Law of Large Numbers (WLLN)

Statement: X̄n = (1/n) ∑Xi   X̄n →P μ

Meaning: Sample mean converges to true mean (in probability).

Example: Coin tosses → fraction of heads converges to true probability.

2. Central Limit Theorem (CLT)

Setup: Xi: i.i.d. with mean μ, variance σ².

Standardized Sum: Yn = (1/√n) ∑i=1n (Xi − μ)
Result: Yn →d N(0, σ²)

Meaning: Sum (or average) becomes normally distributed.

Importance: Explains why the normal distribution appears universally.

Real-Life Example: Noise in electrical current = sum of many small independent effects → Gaussian.

Practical Insight: Even for n ≈ 10–20, distribution already looks bell-shaped.

Final Summary
Tail Bounds: Markov → uses mean. Chebyshev → uses variance. Hoeffding → exponential decay (strongest).

Convergence: In probability → values stabilize. In distribution → shape stabilizes.

Limit Laws: WLLN → sample mean → true mean. CLT → sum → normal distribution.

Key Insight: More data ⇒ less uncertainty. Randomness averages out → predictable patterns emerge.
