Lecture 1: Constrained Optimization (Part 1)

Single Inequality Constrained Optimization

Problem Definition: We want to minimize a function: f(x)

subject to a constraint: g(x) ≤ 0

This is called a single inequality constrained optimization problem. It is very important in machine learning (e.g., regularization, SVM).

Why Not Simple Gradient Descent?

  • Gradient descent works for unconstrained problems.
  • Here, we must also respect the boundary g(x) ≤ 0.
  • Therefore, we need special optimality conditions.

Verifying if a Point is Optimal

Given a candidate solution x*:

1. Feasibility Check

g(x*) ≤ 0

If g(x*) > 0, the point is not feasible → not optimal.

2. Necessary Condition for Optimality

At the optimal point, there should be no direction that both:

  • reduces the function, and
  • satisfies the constraint.

Core Concepts: Directions

(A) Descent Direction: A direction d that reduces the objective function:

dT∇f(x*) < 0

Meaning: moving along d decreases f(x).

(B) Feasible Direction: A direction d that keeps us inside the constraint:

dT∇g(x*) < 0

Meaning: moving along d keeps g(x) ≤ 0.

Key Idea

If a direction satisfies both: dT∇f(x*) < 0 and dT∇g(x*) < 0, then:

  • We can reduce f(x)
  • And still satisfy the constraint

Thus, x* is not optimal.

Geometric Intuition

  • Descent directions form a half-space.
  • Feasible directions form another half-space.
  • Their intersection forms a cone of directions that are both feasible and descent.
  • If such a cone exists → x* is not optimal.
  • If no intersection exists → x* can be optimal.

Gradient Relation Insight

At the optimal point, ∇f(x*) and ∇g(x*) must be arranged such that no common direction reduces both. This leads to the Karush-Kuhn-Tucker (KKT) conditions.

Example
Problem: Minimize f(x) = x², Subject to x ≥ 1 (or g(x) = 1−x ≤ 0)

Step 1: Feasible Region — Allowed values: x ≥ 1.
Step 2: Check Minimum — Without constraint → minimum at x = 0 (not feasible). With constraint → smallest allowed value is x* = 1.
Step 3: Check Directions — Moving left decreases f(x) but violates constraint. No direction exists that satisfies both.

Thus, x* = 1 is optimal.
Final Summary
To check optimality:
  1. Feasibility: g(x*) ≤ 0
  2. No common descent + feasible direction.
Key formulas: Descent: dT∇f(x*) < 0 | Feasible: dT∇g(x*) < 0
If both are satisfied → not optimal. If impossible → optimal.

Lecture 2: Constrained Optimization (Part 2)

Core Objective

Goal: Find optimal point x* that:

  • Minimizes f(x)
  • Satisfies constraint g(x)

Optimal point definition: No direction exists that is both a descent direction, and a feasible direction.

Inequality Constraints (g(x) ≤ 0)

Gradient Relationship: Optimality depends on relation between:

  • ∇f(x*): objective gradient
  • ∇g(x*): constraint gradient

(A) Suboptimal Case (Parallel Gradients)

If ∇f(x*) and ∇g(x*) point in the same direction:

  • Every descent direction is also feasible
  • Many directions reduce f(x)
  • Not optimal

(B) Optimal Case (Anti-parallel Gradients)

If ∇f(x*) and ∇g(x*) point in opposite directions:

  • Any descent direction violates constraint
  • No feasible descent direction exists
Mathematical Condition: ∇f(x*) = −λ∇g(x*), λ > 0

Here, λ is the Lagrange multiplier and must be positive.

Equality Constraints (g(x) = 0)

Feasibility Condition: To stay on constraint surface:

∇g(x*)Td = 0

Direction d must be perpendicular to gradient of constraint.

Optimality Idea

  • Movement restricted to a surface/line
  • Along that surface: no descent possible
Mathematical Condition: ∇f(x*) = −λ∇g(x*)

Here, λ can be positive or negative, but must be non-zero.

Lagrange Multipliers

  • Definition: λ is called a Lagrange Multiplier (named after Joseph-Louis Lagrange).
  • Purpose: Converts constrained problems into equations: ∇f(x*) = −λ∇g(x*)
  • Used to find candidate optimal points.

Key Differences: Inequality vs Equality

FeatureInequality (g(x) ≤ 0)Equality (g(x) = 0)
Feasible directionsInside regionAlong surface only
Gradient relationAnti-parallelParallel or anti-parallel
Lagrange multiplierλ > 0Any non-zero
Movement freedomRestricted by boundaryRestricted to surface

Simple Examples

Example 1 (Inequality)
Minimize: f(x) = x² — Subject to: x ≥ 1 ⇒ g(x) = 1−x ≤ 0
  • Unconstrained minimum: x = 0 (not feasible).
  • With constraint: smallest allowed value is x* = 1.
  • At x*, gradients are opposite → satisfies condition.
  • Optimal point = 1
Example 2 (Equality)
Minimize: f(x,y) = x² + y² — Subject to: x + y = 1

Step 1: Constraint — g(x,y) = x + y − 1 = 0
Step 2: Apply condition — ∇f = (2x,2y), ∇g = (1,1); ∇f = −λ∇g ⇒ 2x = −λ, 2y = −λ ⇒ x = y
Step 3: Use constraint — x + y = 1 ⇒ 2x = 1 ⇒ x = y = 1/2

Final Answer: (x*, y*) = (½, ½)
Final Summary
Optimal point: No feasible descent direction.
Inequality: Gradients anti-parallel, λ > 0.
Equality: Gradients parallel/anti-parallel, λ ≠ 0.

Master Formula: ∇f(x*) = −λ∇g(x*)

Lecture 3: Method of Lagrange Multiplier, Projected Gradient Descent

Method of Lagrange Multipliers

Problem Type: Used for equality constrained optimization: min f(x) subject to g(x) = 0

Necessary Conditions for Optimality

1. Feasibility Condition

g(x*) = 0

Solution must satisfy the constraint.

2. Gradient Alignment Condition

Gradients must be parallel or anti-parallel:

∇f(x*) = −λ∇g(x*)

where λ = Lagrange Multiplier (can be positive or negative).

Step-by-Step Solving Method

  1. Find gradients: ∇f(x), ∇g(x)
  2. Apply condition: ∇f = −λ∇g
  3. Add constraint: g(x) = 0
  4. Solve system of equations
  5. Check candidate points:
    • Minimum → smallest f(x)
    • Maximum → largest f(x)
    • Others → saddle points
Example
Problem: f(x,y) = x² + y², subject to x + y = 1

Step 1: Constraint — g(x,y) = x + y − 1 = 0
Step 2: Gradients — ∇f = (2x, 2y), ∇g = (1,1)
Step 3: Apply condition — (2x, 2y) = −λ(1,1) ⇒ x = y
Step 4: Use constraint — x + y = 1 ⇒ x = y = ½

Final Answer: (x*, y*) = (½, ½)

Limitation: Hard to solve complex/nonlinear equations analytically.

Projected Gradient Descent (PGD)

Idea: Used when analytical solution is difficult. Combines:

  • Gradient Descent (optimization)
  • Projection (constraint satisfaction)

Algorithm Steps

  1. Initialization: Start with x0.
  2. Gradient Step: x't+1 = xt − η∇f(xt)
  3. Projection Step: xt+1 = ΠS(x't+1)

Ensures constraint satisfaction.

Projection Operator

Finds closest feasible point:

ΠS(x) = arg miny∈S ‖x − y‖²
Example:
Constraint: x ≥ 0.
After gradient step: x′ = −3.
Projection: closest feasible point = 0.
So, x = 0.

Key Requirement

Projection must be:

  • Fast
  • Ideally polynomial time

Convexity & Performance

  • Convex Set: If constraint set S is convex → PGD guaranteed to converge.
  • Convex Function: If f(x) is convex → faster convergence, better performance.

Importance in Machine Learning: Many ML problems have convex functions and convex constraints → PGD works efficiently.

Quick Comparison

MethodUse CaseType
Lagrange MultipliersExact solutionAnalytical
PGDLarge/complex problemsIterative
Final Summary
Lagrange Method: Solve ∇f = −λ∇g. Works for equality constraints.
PGD: Step → Project → Repeat. Works for complex constraints.
Convexity: Guarantees correctness and speed.

Lecture 4: Introduction to Convexity

Motivation for Studying Convexity

  • Convexity is crucial in machine learning because optimization is everywhere.
  • We aim to minimize loss/error.
  • Key Idea:
    • General functions → may have many local minima.
    • Convex functions → have only one global minimum.

Conclusion: Convexity guarantees we can find the best possible solution.

Definition of a Convex Set

A set S ⊆ Rd is convex if: for any two points in the set, the entire line segment between them also lies in the set.

Mathematical Condition: λx1 + (1−λ)x2 ∈ S, ∀ x1, x2 ∈ S, λ ∈ [0,1]

Understanding λ

  • λ = 1 → point = x1
  • λ = 0 → point = x2
  • λ = 0.5 → midpoint

Think of λ as a slider moving between two points.

Visual Intuition

  • Convex Sets: Any line drawn between two points stays inside.
  • Non-Convex Sets: Line may go outside the set.

Examples of Convex Sets

(A) Intervals (1D): Example: [a,b] → always convex. Union like [1,2]∪[4,5] → not convex.

(B) Geometric Shapes: Circle, ellipse (filled), solid polygon → convex.

(C) Hyperplanes: Defined as: wTx = b — Flat surface (line/plane) → always convex.

(D) Half-Spaces: Defined as: wTx ≤ b — One side of a plane → convex. Very important in ML (e.g., classifiers).

Non-Convex Sets

Examples: Disjoint sets (with gaps), dented or hollow shapes.

Reason: Line between two points goes outside the set.

Proof Sketch: Hyperplane is Convex
Given: S = {x ∣ wTx = b}
Step 1: Take two points: wTx1 = b, wTx2 = b
Step 2: Check line segment: wT(λx1 + (1−λ)x2)
Step 3: Expand: λwTx1 + (1−λ)wTx2
Step 4: Substitute: λb + (1−λ)b = b
Conclusion: New point also satisfies equation → set is convex.

Why Convexity Matters

  • Guarantees global optimum.
  • Makes optimization easy and reliable.
  • Used in: Linear Regression, SVM, Logistic Regression
Final Summary
Convex set: Line between any two points lies inside.
Key formula: λx1 + (1−λ)x2 ∈ S
Convex = no dents, no gaps.
Important convex sets: intervals, circles, hyperplanes, half-spaces.

Lecture 5: Properties of Convex Sets

Intersection of Convex Sets

Key Property: The intersection of convex sets is always convex.

Mathematical Idea: If S1 and S2 are convex, then: S = S1 ∩ S2 is also convex.

Intuition

  • Take any two points in the intersection.
  • They belong to both sets.
  • The line between them stays in both sets.
  • Therefore, it stays in the intersection.
Example: Linear System Ax = b
Each equation corresponds to a hyperplane (convex). Solution set = intersection of hyperplanes → convex.

Convex Combinations

Definition: A point formed by weighted average of points.

z = ∑i=1n λixi,   λi ≥ 0,   ∑i=1n λi = 1

Key Conditions

  • Weights λi ≥ 0
  • Sum of weights = 1
Example: z = 0.1x1 + 0.3x2 + 0.5x3 + 0.1x4 — Valid convex combination. Some weights can be zero.

Intuition: Mixing points like a weighted average.

Convex Hull

Definition 1 (Combination View): Set of all convex combinations:

Conv(S) = {∑λixi}

Definition 2 (Intersection View): Smallest convex set containing all points. Equivalent to intersection of all convex sets containing the points.

Key Properties

  • Convex hull is always convex.
  • Both definitions are equivalent.

Visual Intuition: Stretch a rubber band around points → shape formed = convex hull.

Example: Points in plane → convex hull = smallest polygon enclosing them.

Euclidean Balls

Definition: Set of all points within a radius.

Formula: ‖x‖ ≤ θ
Norm Definition: ‖x‖ = √(∑i=1d xi²)

Intuition: All points inside a circle (2D) or sphere (3D). Always convex.

Why These Concepts Matter in ML

  • Convex hull: used in clustering, geometry.
  • Intersection: used in constraints.
  • Euclidean balls: used in regularization (L2 norm).
  • Convex combinations: used in mixtures, probability.

Practice Questions

Q1: Convex Hull is Convex
Take two convex combinations. Combine again → still convex combination. Hence convex.

Q2: Two Definitions Equivalent
Convex combinations ⊆ smallest convex set. Smallest convex set ⊆ convex combinations. Hence equal.

Q3: Euclidean Ball is Convex
Use: ‖λx1 + (1−λ)x2‖ ≤ θ. Triangle inequality ensures convexity.
Final Summary
Intersection of convex sets → convex.
Convex combination → weighted average.
Convex hull → smallest convex set.
Euclidean ball → all points within radius.

Lecture 6: Convex Functions

Introduction & Importance

  • Convex functions are very important in machine learning.
  • Loss/error functions are often convex, making optimization easy and reliable.
  • Key Advantage: Convex function ⇒ only one global minimum.
  • No risk of getting stuck in local minima.

Important Condition: A function is convex only if its domain is a convex set. Example: Rd is convex → valid domain.

Definition 1: Epigraph (Geometric View)

Definition: A function is convex if its epigraph is convex.

epi(f) = {(x, z) ∣ z ≥ f(x)}

Intuition: All points on or above the graph. If this region is convex → function is convex.

Definition 2: Linear Interpolation (Most Important)

f(λx1 + (1−λ)x2) ≤ λf(x1) + (1−λ)f(x2)

Meaning: Value at midpoint ≤ line joining endpoints.

Intuition: Convex → graph lies below the line. Non-convex → graph goes above the line.

Definition 3: First-Order Condition (Gradient)

f(y) ≥ f(x) + (y−x)T∇f(x)

Meaning: Tangent line (linear approximation) is always below the function.

Interpretation: Gradient gives a supporting hyperplane. Function never dips below it.

Definition 4: Second-Order Condition (Hessian)

Formula: A function is convex if its Hessian matrix H is Positive Semi-Definite (PSD).

Hessian Definition: Hij = ∂²f / ∂xi∂xj

PSD Condition: All eigenvalues ≥ 0.

Intuition: Positive curvature → convex shape. No downward bending.

Convex Example: f(x) = x²
First derivative: f′(x) = 2x. Second derivative: f″(x) = 2. Since 2 > 0, function is convex.

Non-Convex Example: f(x) = −x²
Second derivative = −2 < 0. Curve bends downward → not convex.

Summary of All Definitions

DefinitionIdea
EpigraphRegion above graph is convex
InterpolationLine above graph
First-orderTangent below function
Second-orderHessian ≥ 0
Final Summary
Convex function ⇒ global minimum guaranteed.
Must satisfy any one definition.
Most used: Interpolation (basic), Hessian (advanced).

Key Formula: f(λx1 + (1−λ)x2) ≤ λf(x1) + (1−λ)f(x2)

Lecture 7: Properties of Convex Functions

Central Theorem: If a function f is convex: Every local minimum is also a global minimum. This is the most powerful property of convex functions.

Definitions of Minima

(A) Local Minimum: A point x* is a local minimum if:

  • There exists a small neighborhood (ball of radius δ)
  • Such that: f(x*) ≤ f(z), ∀ z near x*

Intuition: Small movements increase function value.

(B) Global Minimum: A point x* is a global minimum if: f(x*) ≤ f(x), ∀ x in domain

Intuition: Best value in entire space.

Key Difference

TypeRegion
Local MinimumSmall neighborhood
Global MinimumEntire domain
Proof Idea (By Contradiction)
Step 1: Assume Opposite — Suppose x* is a local minimum but not global. Then there exists z such that: f(z) < f(x*)
Step 2: Use Local Property — Around x*, small ball exists: f(x*) ≤ f(y)
Step 3: Construct Point — Take point on line: y = λx* + (1−λ)z. Choose λ small → y close to x*.
Step 4: Apply Convexity — f(y) ≤ λf(x*) + (1−λ)f(z)
Step 5: Use Assumption — Since f(z) < f(x*), we get: f(y) < f(x*)

Contradiction: Local minimum requires f(x*) ≤ f(y). Thus assumption is wrong.
Conclusion: Local minimum = Global minimum.

Visual Intuition

  • Convex Function: Only one minimum → global.
  • Non-Convex Function: Many local minima → confusing.
Convex Function: f(x) = x² — Only one minimum at x = 0. Local = Global.

Non-Convex Function: f(x) = x⁴ − x² — Multiple local minima, only one global.

Practical Importance in ML

  • Optimization Made Easy: For convex functions, finding any local minimum is enough → automatically global optimum.
  • Gradient Descent: Finds local minima. For convex functions, gradient descent ⇒ global minimum.
  • Why ML Loves Convexity: Guarantees correctness, easier optimization, faster convergence.
Final Summary
Convex function ⇒ no bad local minima.
Key theorem: Local minimum = Global minimum.
Proof idea: Assume contradiction → use convexity inequality → reach impossible condition.

Golden takeaway: In convex optimization, you can trust your solution.
