Week 10: Short Notes — Machine Learning Foundations

Lecture 1: Properties of Convex Functions

Core Concepts

Local vs Global Minima

For a convex function: Every local minimum = global minimum. This is the key property: no "bad" local minima.

Non-uniqueness of Minima

  • A convex function can have multiple global minima.
  • All such minima form a convex set.
  • In 1D (R → R): minima form an interval [a,b].
Example: A flat-bottom function between a and b. Every point in [a,b] is optimal.

Concave Functions

A function g(x) is concave if: f(x) = −g(x) is convex.

Example: g(x) = −x² ⟹ concave.

First-Order Optimality Condition

Theorem: For convex, differentiable f: Rd → R:

x* is global minimum ⟺ ∇f(x*) = 0

Necessity (⇒)

If x* is minimum:

  • Gradient must be zero: ∇f(x*) = 0.
  • Reason: If gradient ≠ 0, moving in direction −∇f(x*) decreases function value → contradiction.

Sufficiency (⇐)

  • If ∇f(x*) = 0, then using convexity: f(y) ≥ f(x) + ∇f(x)T(y−x)
  • At x = x*: f(y) ≥ f(x*)
  • So, x* is global minimum.

Algorithm Insight: In convex problems, gradient descent reaching ∇f = 0 ⇒ global optimum found.

Properties of Convex Functions

Property 1: Sum Rule

If f and g are convex: h(x) = f(x) + g(x) is convex.

Property 2: Composition Rule

If:

  • g(x) is convex
  • f(x) is convex and non-decreasing

Then: h(x) = f(g(x)) is convex

Non-decreasing condition: x ≥ y ⟹ f(x) ≥ f(y)

Property 3: Convex + Linear

If:

  • f is convex
  • g is linear

Then: h(x) = f(g(x)) is convex (no need for non-decreasing condition here)

Examples & Counterexamples

Non-convex Composition: Even if both functions are convex:
  • g(x) = x² (convex)
  • f(x) = e−x (convex)
Then: h(x) = f(g(x)) = e−x² — This is not convex (Gaussian curve).
Multiple Minima Example
Function constant in interval [a,b], increases outside. All points in [a,b] are global minima.
Final Quick Summary
  • Convex ⇒ local min = global min
  • Multiple minima ⇒ form convex set (interval in 1D)
  • Optimality condition: ∇f(x*) = 0
  • Key rules:
    • Sum of convex → convex
    • Convex ∘ convex (with non-decreasing) → convex
    • Convex ∘ linear → convex
  • Be careful: Convex ∘ convex ≠ always convex

Lecture 2: Applications of Optimization in Machine Learning

Problem Setup

Goal: Predict a real-valued output using input features. Learn a function: h: Rd → R

Data Representation: Training data: (xi, yi), i = 1,2,...,n

  • Features: xi ∈ Rd
  • Output: yi ∈ R

Linear Model: Assume linear relationship: h(x) = wTx, w ∈ Rd

Example: Features = (height, weight, BMI). Model predicts salary/score using weighted sum.

Performance Measure (SSE)

Sum of Squared Errors: Measures prediction error:

f(w) = ∑i=1n (wTxi − yi)²

Standard Optimization Form: minw∈Rd (1/2) ∑i=1n (wTxi − yi)²

Reason for ½: Simplifies derivative (cancels factor of 2).

Convexity of SSE

Key Idea: Convex ⇒ global minimum guaranteed.

Step-by-step Reasoning

Each term: hi(w) = (wTxi − yi)² is convex because:

  • z² is convex
  • wTxi − yi is linear
  • Convex ∘ Linear = Convex

Sum of convex functions: f(w) = ∑ hi(w) is also convex.

Conclusion: SSE is convex → no local minima problem.

Matrix Formulation

Definitions

  • X ∈ Rn×d: data matrix
  • y ∈ Rn: output vector
Compact Form: f(w) = (1/2)‖Xw − y‖&sub2;²   or   f(w) = (1/2)(Xw − y)T(Xw − y)

Gradient & Closed-Form Solution

Gradient: ∇f(w) = XTXw − XTy

Optimal Solution (Normal Equation)

Set gradient = 0:

XTXw* = XTy
w* = (XTX)−1XTy

Special Case: If XTX is not invertible → use pseudo-inverse. Example: redundant features → singular matrix.

Computational Issues

Problem with Closed Form: Matrix inverse cost: O(d³)

Example: Image with 1 million features → infeasible computation.

Gradient Descent (GD)

Idea: Iteratively update weights instead of solving directly.

Update Rule: wt+1 = wt − ηt(XTXwt − XTy)   where ηt = learning rate.

Advantages

  • Avoids matrix inversion
  • Works for large datasets

Stochastic Gradient Descent (SGD)

Problem with GD: Uses full dataset → slow for large n.

SGD Idea: Use random sample instead of full data.

Update (conceptual): w ← w − η · gradient from few samples

Key Insight

Each step is noisy, but over time:

  • Errors cancel out
  • Converges to optimal w*

Example: Dataset = billions of points. SGD uses mini-batches (e.g., 32, 64 samples).

Final Quick Summary
Model: h(x) = wTx | Loss: ½ ∑(wTxi − yi)²
Convexity: Global optimum guaranteed. Solution: w* = (XTX)−1XTy
Issues: Inversion expensive (O(d³)).
Alternatives: Gradient Descent → iterative, Stochastic Gradient Descent → scalable

Lecture 3: Revisiting Constrained Optimization

Review: Unconstrained Optimization

Key Idea (Convex Case):

  • If f is convex: local minimum = global minimum.
  • First-Order Optimality Condition: ∇f(x*) = 0 ⇒ x* is global optimum

Algorithm: Use Gradient Descent: iteratively move towards ∇f(x) = 0.

Constrained Optimization Problem

  • Standard Form: minx∈Rn f(x) subject to h(x) ≤ 0
  • Goal: Find optimal x that minimizes f(x) while satisfying h(x) ≤ 0.
  • Example: Minimize cost subject to budget ≤ limit, resource usage ≤ capacity.

Lagrangian Function

  • Definition: L(x,λ) = f(x) + λh(x)
  • Components: x: decision variable & λ ≥ 0: Lagrange multiplier

Intuition: Converts constrained problem into unconstrained form.

Case 1 (Constraint satisfied)

h(x) ≤ 0 ⟹ λh(x) ≤ 0. Maximum occurs at λ = 0.

maxλ≥0 L(x,λ) = f(x)

Case 2 (Constraint violated)

h(x) > 0 ⟹ λh(x) > 0.

As λ → ∞, maxλ≥0 L(x,λ) = ∞

Insight: Invalid points get infinite penalty; valid points retain original value.

Primal Problem

  • Reformulation: minx maxλ≥0 L(x,λ)
  • Logic: If constraint violated → value = ∞. Minimization automatically selects feasible points.
  • Output: Optimal solution: x*. Optimal value: f(x*).

Dual Problem

  • Definition (swap order): maxλ≥0 minx L(x,λ)
  • Dual Function: g(λ) = minx[f(x) + λh(x)]
  • Interpretation: For each λ, solve an unconstrained minimization.

Properties of Dual Problem

Easier Optimization: Inner problem minxL(x,λ) is unconstrained → easier to solve.

Concavity: g(λ) is always concave, even if f is not convex.

Dual Problem: maxλ≥0 g(λ)

Concave maximization → well-behaved.

Primal vs Dual Relationship

Important Concept: Duality Gap

minx maxλ L(x,λ) ≠ maxλ minx L(x,λ)

Duality Gap: Difference between primal optimal value and dual optimal value.

Key Insight: Gap = 0 → strong duality, Gap ≠ 0 → weak duality

Intuition Summary

  • Lagrangian: Adds penalty for constraint violation.
  • Primal: Focuses on feasible solutions.
  • Dual: Converts problem into easier unconstrained form.
  • Dual provides: Lower bound on primal solution.
Final Quick Summary
Problem: min f(x) s.t. h(x) ≤ 0 | Lagrangian: L(x,λ) = f(x) + λh(x)
Primal: minx maxλ L(x,λ) | Dual: maxλ minx L(x,λ)
Dual Function: g(λ) = minxL(x,λ)
Memory Trick: "Primal = min then max, Dual = max then min."

Lecture 4: Relation between Primal and Dual Problem, KKT Conditions

Primal vs Dual Problem

  • Lagrangian Function: L(x,λ) = f(x) + λh(x), λ ≥ 0
  • Primal Problem: minx maxλ≥0 L(x,λ) — Focus: feasible solutions (h(x) ≤ 0).
  • Dual Problem: maxλ≥0 minx L(x,λ) — Focus: optimize over multipliers λ.

Auxiliary Function J(x)

J(x) = f(x) if h(x) ≤ 0;   J(x) = ∞ if h(x) > 0

Key Insight: Converts constrained → unconstrained problem. Penalizes infeasible points with ∞.

Important Relation: L(x,λ) ≤ J(x), ∀ x, λ ≥ 0 — Lagrangian provides a lower bound.

Duality Theorems

  • Weak Duality (Always True): g(λ*) ≤ f(x*) — Dual optimum ≤ Primal optimum. Meaning: dual gives a lower bound.
  • Strong Duality: g(λ*) = f(x*) — Holds if f(x) is convex, h(x) is convex, and regularity conditions are satisfied.

Importance: solve dual instead of primal, same optimal value but often easier.

KKT Conditions

A. Single Inequality Constraint

Problem: min f(x) s.t. h(x) ≤ 0

KKT Conditions:

  1. Stationarity: ∇f(x*) + λ*∇h(x*) = 0
  2. Complementary Slackness: λ*h(x*) = 0 — Meaning: either constraint active (h(x*) = 0) or multiplier zero (λ* = 0).
  3. Primal Feasibility: h(x*) ≤ 0
  4. Dual Feasibility: λ* ≥ 0

B. Multiple Constraints (General Case)

  • Problem: Inequality: hi(x) ≤ 0, Equality: lj(x) = 0
  • Lagrangian: L(x,u,v) = f(x) + ∑i=1m uihi(x) + ∑j=1n vjlj(x)

General KKT Conditions:

  1. Stationarity: ∇f(x*) + ∑ui*∇hi(x*) + ∑vj*∇lj(x*) = 0
  2. Complementary Slackness: ui*hi(x*) = 0, ∀ i
  3. Primal Feasibility: hi(x*) ≤ 0, lj(x*) = 0
  4. Dual Feasibility: ui* ≥ 0

Key Results

  • Necessity: If optimal → must satisfy KKT (under conditions).
  • Sufficiency:
    • Convex case → global optimum.
    • General case → local optimum.

Intuition

  • Multipliers λ: measure importance of constraints.
  • Complementary slackness: either constraint is active or ignored.
  • KKT = balance condition: gradient of objective + constraints = 0.
Final Quick Summary
Primal: minx maxλ L(x,λ) | Dual: maxλ minx L(x,λ)
Weak Duality: g(λ*) ≤ f(x*) | Strong Duality: g(λ*) = f(x*)
KKT Conditions: Stationarity, Complementary Slackness, Primal Feasibility, Dual Feasibility
Memory Trick: "Stationary + Feasible + Slackness = Optimal (KKT)."

Lecture 5: KKT Conditions Continued

Core Concepts

Constrained Optimization in ML: Many ML problems are formulated as:

min f(x) subject to constraints

Example: classification with margin constraints, resource or boundary restrictions in models.

Convexity: In ML, most objective functions are convex. Property: convex ⇒ local optimum = global optimum.

Strong Duality: If the objective function and constraints are convex: min max = max min

Importance: allows solving the dual instead of the primal, often easier and more efficient.

Dual Problem – Benefits: Easier to solve, enables non-linear models, kernel methods, and works well in high dimensions.

KKT Conditions: Provide optimality conditions. For convex problems: necessary and sufficient.

Key Formulae (SVM Focus)

Objective Function (Quadratic): minw (1/2)‖w‖²

Minimizes magnitude of weight vector → leads to maximum margin.

Expanded Form: ‖w‖² = ∑i=1d wi²

Linear Constraints: yi(wTxi) ≥ 1, ∀ i

Interpretation:

  • If yi = +1: wTxi ≥ 1
  • If yi = −1: wTxi ≤ −1

Ensures correct classification with margin.

Key ML Examples

Support Vector Machine (SVM):
  • Type: constrained optimization problem
  • Structure: objective quadratic (convex), constraints linear
  • Key idea: find hyperplane maximizing margin between classes
  • Example: classify emails (Spam +1 vs Not Spam −1)
Linear Regression:
  • Type: unconstrained optimization
  • Objective: min ∑(wTxi − yi)²
  • No constraints involved.

Optimization Flow (Big Picture)

Directional Movement: Move in direction of −∇f(x) → gradient descent.

Lagrangian Formulation: Add constraints using: L(x,λ) = f(x) + λh(x)

Primal → Dual: Use duality theory and KKT conditions to transform problem into easier form.

Intuition Summary

  • Convexity → no local traps.
  • Dual problem → easier computation.
  • KKT → bridge between primal and dual.
  • SVM → practical example of all concepts.
Final Quick Summary
ML problems → often constrained optimization.
Convex ⇒ global optimum guaranteed.
Strong Duality: min max = max min

SVM: Objective: minimize ‖w‖² | Constraint: yiwTxi ≥ 1
Tools: Gradient Descent, Lagrangian, KKT conditions.
Memory Trick: "Convex + KKT + Duality = Solve ML Optimization Easily."
