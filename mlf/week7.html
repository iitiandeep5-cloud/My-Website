Week 7: Short Notes

MLF — Principal Component Analysis (PCA)

Lecture 1: Principal Component Analysis

What is PCA?

  • PCA (Principal Component Analysis) is a:
    • Dimensionality reduction technique.
    • Feature selection/extraction method.
  • Converts high-dimensional data into lower-dimensional data while preserving important information.

Core Objective

  • Reduce dimensions ($d \to m$, where $m < d$).
  • While: keeping maximum information & reducing redundancy and noise.

Two Equivalent Viewpoints

(A) Minimize Reconstruction Error

  • Project data so that projected data $\approx$ original data.
  • Goal: minimum information loss.

(B) Maximize Variance

  • Choose directions where data spreads the most.
  • Goal: maximum variance retained.

Problem Setup

  • Dataset: $D = \{x_1, x_2, \ldots, x_n\}$, $x_i \in \mathbb{R}^d$
  • Input: $m$ = target dimension, $m < d$.
  • Goal: Find best $m$-dimensional subspace.

Basis Representation

  • Choose orthonormal basis: $B = \{u_1, u_2, \ldots, u_m\}$
  • Extend to full basis: $B' = \{u_1, \ldots, u_m, u_{m+1}, \ldots, u_d\}$

Representation of Data

Each data point:

$$x_i = \sum_{j=1}^{d} (x_i^T u_j) u_j$$

Coefficient: $x_i^T u_j$ = projection of $x_i$ on $u_j$

Projection (Approximation)

Projected vector:

$$\tilde{x}_i = \sum_{j=1}^{m} z_{ij} u_j + \sum_{j=m+1}^{d} \beta_j u_j$$

Optimal values: $z_{ij} = x_i^T u_j$,   $\beta_j = \bar{x}^T u_j$

Data Centering

Mean of data:

$$\bar{x} = \frac{1}{n} \sum_{i=1}^{n} x_i$$

If centered ($\bar{x} = 0$):

  • Simplifies calculations.
  • Second term disappears.

Covariance Matrix

$$C = \frac{1}{n} \sum_{i=1}^{n} (x_i - \bar{x})(x_i - \bar{x})^T$$

Meaning:

  • Measures spread and relationships between features.
  • Core matrix used in PCA.

Optimization Goal

Minimize reconstruction error: $J$ = Average Squared Error

Optimal error:

$$J^* = \sum_{j=m+1}^{d} u_j^T C u_j$$

Geometric Intuition

  • Data is projected onto a line/plane.
  • Best direction = maximum spread.
  • First principal component = direction of maximum variance.

Key Insight

PCA finds:

  • Best $m$ directions ($u_1, u_2, \ldots, u_m$).
  • That maximize variance and minimize reconstruction error.

Example (Simple Intuition)

  • Suppose data has 2 features: Height & Weight.
  • Observations: highly correlated.
  • PCA converts them into:
    • PC1: main direction of variation.
    • PC2: less important direction.
  • We can drop PC2 → reduce dimension from 2 → 1.

Final Conclusion

  • PCA = Projection + Optimization.
  • Converts data into new axes (principal components).
  • Keeps: maximum variance, minimum information loss.

Quick Revision Summary

  • PCA reduces dimension ($d \to m$).
  • Two views: minimize error / maximize variance.
  • Key matrix: covariance matrix $C$.
  • Output: new basis vectors (principal components).
  • Works best on centered data.

Lecture 2: Principal Component Analysis (Contd.)

Core Objective of PCA

  • Goal: Find the best low-dimensional subspace ($m < d$) such that reconstruction error is minimized.
  • PCA reduces dimensions by selecting important eigenvalues and eigenvectors.

PCA as an Optimization Problem

  • Problem type: Constrained Optimization.
  • We want unit vectors ($\|u\| = 1$) that minimize reconstruction error.
Lagrangian Formulation: $L(u, \lambda) = u^T C u - \lambda(u^T u - 1)$
After solving: $Cu = \lambda u$
This is the Eigenvalue Equation: $u \to$ eigenvector, $\lambda \to$ eigenvalue

Properties of Covariance Matrix ($C$)

  • Size: $d \times d$.
  • Type: Real and symmetric.

Important Properties:

  • All eigenvalues are real.
  • Eigenvectors are orthonormal.
  • Eigenvalues can be ordered: $\lambda_1 \geq \lambda_2 \geq \cdots \geq \lambda_d$

Minimizing Reconstruction Error

Error expression:

$$J^* = \sum_{j=m+1}^{d} u_j^T C u_j$$

Key Idea:

  • Small eigenvalues → less important directions.
  • Large eigenvalues → important directions.

Selection Rule:

  • Keep: Top $m$ eigenvectors (largest eigenvalues).
  • Discard: Remaining (small eigenvalues).

This minimizes error.

PCA Algorithm (Step-by-Step)

Steps

Step 1: Compute Mean $$\bar{x} = \frac{1}{n} \sum_{i=1}^{n} x_i$$ Step 2: Compute Covariance Matrix $$C = \frac{1}{n} \sum_{i=1}^{n} (x_i - \bar{x})(x_i - \bar{x})^T$$ Step 3: Eigen Decomposition — find eigenvalues ($\lambda$) and eigenvectors ($u$).

Step 4: Select Top $m$ Components — choose eigenvectors with largest eigenvalues.

Step 5: Project Data — $\tilde{x}_i = $ Projection onto selected eigenvectors

Geometric Intuition

  • Data is rotated to new axes.
  • First axis = maximum variance direction.
  • Projection reduces dimension.

Example Insight

  • Case: Data lies on a line in 2D space.
  • PCA finds that exact line.
  • Reconstruction error = 0.
  • Best 1D subspace: eigenvector with largest eigenvalue.

Alternative Viewpoint

  • PCA can also be seen as Maximizing Variance.
  • Choose directions where variance is maximum.
  • Equivalent to minimizing reconstruction error.

Quick Revision Summary

  • PCA = Eigenvalue problem.
  • Solve: $Cu = \lambda u$
  • Keep: Top $m$ eigenvectors (largest $\lambda$).
  • Steps: Mean → Covariance → Eigen decomposition → Projection
  • Two views: Minimize error / Maximize variance.

Lecture 3: PCA As Maximizing Variance

Core Perspective

  • PCA can be understood as:
    • Maximizing variance of projected data.
    • Equivalent to minimizing reconstruction error.
  • Idea: Project data onto a direction where it is most spread out.

Data Setup

Dataset: $D = \{x_1, x_2, \ldots, x_n\}$, $x_i \in \mathbb{R}^d$

Mean: $$\bar{x} = \frac{1}{n} \sum_{i=1}^{n} x_i$$

Projection of Data

Project point $x_i$ onto direction $u$: $\text{Projection} = (x_i^T u)u$

  • $u$ is a unit vector.
  • Gives direction of projection.

Variance Objective

We maximize:

$$\frac{1}{n} \sum_{i=1}^{n} (x_i^T u - \bar{x}^T u)^2$$

Meaning:

  • Measures spread of projected points.
  • Higher variance → better information retention.

Covariance Matrix

Formula: $$C = \frac{1}{n} \sum_{i=1}^{n} (x_i - \bar{x})(x_i - \bar{x})^T$$

Then variance becomes: $u^T C u$

Optimization Problem

Maximize: $u^T C u$
Subject to: $u^T u = 1$

Solution (Eigenvalue Problem)

Using Lagrangian: $Cu = \lambda u$

Key Results: $u$ = eigenvector of $C$.   $\lambda$ = eigenvalue.

Final Insight

  • Maximum variance = largest eigenvalue.
  • Best direction = eigenvector of largest eigenvalue.

Extension to $m$ Dimensions

Instead of one direction:

  • Choose top $m$ eigenvalues and their corresponding eigenvectors.

Terminology:

  • Principal Directions → eigenvectors.
  • Principal Components → projected values.

Geometric Intuition

  • Rotate axes to align with data spread.
  • First axis = maximum variance.
  • Second axis = next highest variance.

Example

Data points: $(-1,-1), (0,0), (1,1)$.

Results:

  • Eigenvalues: $\lambda_1 = 4/3$, $\lambda_2 = 0$.
  • Interpretation: Data lies on a line → one direction has all variance.
  • Principal direction: $u_1 = \frac{1}{\sqrt{2}}(1,1)^T$
  • Maximum variance: $= 4/3$

Final Conclusion

PCA:

  1. Maximizes variance.
  2. Minimizes information loss.

Steps:

  1. Compute mean.
  2. Compute covariance matrix.
  3. Find eigenvalues & eigenvectors.
  4. Select top $m$.
  5. Project data.

Quick Revision Summary

  • Objective: $\max u^T C u$
  • Constraint: $u^T u = 1$
  • Solution: $Cu = \lambda u$
  • Choose: Largest eigenvalues → best directions.
  • PCA = Variance maximization + dimensionality reduction.

Lecture 4: PCA In Higher Dimensions

Core Problem (Why Optimization Needed?)

  • Standard PCA issue: covariance matrix size $= d \times d$.
  • When $d \gg n$ (features $\gg$ data points), computation becomes very expensive.
Key Insight: $\text{Rank}(C) \leq n$. We don't need full computation.

Rank Concept

Covariance matrix: $$C = \frac{1}{n} \sum_{i=1}^{n} (x_i - \bar{x})(x_i - \bar{x})^T$$

  • Each term is rank 1.
  • Sum of $n$ such matrices → $\text{Rank}(C) \leq n$.
  • Result: Only $n$ meaningful eigenvalues.
  • Remaining $(d-n)$ eigenvalues $= 0$.
  • No need to compute all $d$ eigenvectors.

Key Matrices

Mean: $$\bar{x} = \frac{1}{n} \sum_{i=1}^{n} x_i$$

Covariance Matrix: $$C = \frac{1}{n} A^T A$$

Matrix $A$ (Centered Data): Size: $A = n \times d$.

$$A = \begin{bmatrix} (x_1 - \bar{x})^T \\ (x_2 - \bar{x})^T \\ \vdots \\ (x_n - \bar{x})^T \end{bmatrix}$$

The PCA "Trick" (Efficiency Idea)

Instead of: $$C = \frac{1}{n} A^T A \quad (d \times d)$$

Use: $$\frac{1}{n} A A^T \quad (n \times n)$$

  • $n \times n$ matrix is much smaller.
  • Huge computational saving when $d \gg n$.

Eigenvalue Relationship

Key result: Non-zero eigenvalues of $\frac{1}{n}A^T A$ and $\frac{1}{n}AA^T$ are the same.

Eigenvector Transformation

If: $$\frac{1}{n} A A^T v_i = \lambda_i v_i$$

Then: $$u_i = A^T v_i$$

  • $u_i$ gives eigenvector of the original covariance matrix.
  • Final relation: $C u_i = \lambda_i u_i$

Step-by-Step Efficient PCA

Steps

  1. Compute mean $\bar{x}$.
  2. Form matrix $A$.
  3. Compute $\frac{1}{n} A A^T$.
  4. Find eigenvalues & eigenvectors $v_i$.
  5. Convert to high dimension: $u_i = A^T v_i$.

Geometric Intuition

  • Instead of working in huge space ($d$-dimensional), work in smaller space ($n$-dimensional).
  • Then map back to original space.

Example (Intuition)

Scenario:
  • Data points $n = 100$.
  • Features $d = 10{,}000$.
  • Normal PCA: work with $10{,}000 \times 10{,}000$ matrix.
  • Efficient PCA: work with $100 \times 100$ matrix.
  • Result: Same principal components, much faster computation.

Final Conclusion

  • When $d \gg n$: use $AA^T$ instead of $A^T A$.
  • Benefits: Faster, memory efficient, same results as standard PCA.

Quick Revision Summary

  • Rank: $\text{Rank}(C) \leq n$.
  • Many eigenvalues $= 0$.
  • Trick: Use $AA^T$ ($n \times n$) instead of $A^T A$ ($d \times d$).
  • Eigenvalues: same for both matrices.
  • Eigenvectors: $u_i = A^T v_i$
