Lecture 1: Goodness of Maximum Likelihood Estimator for Linear Regression

Core Concept: Linear Regression can be viewed as Maximum Likelihood Estimation (MLE).

Model: y = wᵀx + ϵ

  • w: true (unknown) parameter
  • ϵ: noise

Noise Assumption: ϵ ∼ N(0, σ²)

Conditional Distribution: y ∣ x ∼ N(wᵀx, σ²)

Important Insight

  • Since y is random → the estimator ŵML is also random.
  • Meaning:
    • Different datasets → different estimates
    • Depends on noise in data

Measuring "Goodness" of Estimator

  • Goal: Measure how close ŵML is to true w.
  • Measure Used: E[‖ŵML − w‖²] called Mean Squared Error (MSE) of estimator

Intuition: Imagine repeating experiment many times → take average error between estimated and true w.

Key Formulae

  • MLE Solution: ŵML = (XXᵀ)⁻¹Xy (Use pseudo-inverse if needed)
  • Goodness Formula: E[‖ŵML−w‖²] = σ² · trace((XXᵀ)⁻¹)

Understanding the Formula

Error depends on two factors:

  1. Noise Variance (σ²):
    • Represents randomness in data
    • Higher noise → worse estimation
    • Cannot control this
  2. Feature Structure (trace((XXᵀ)⁻¹)):
    • Depends on feature quality & correlation
    • Good features → smaller trace → better estimator
    • Poor features → larger error
Example: House Price Prediction
  • Case 1: Low Noise — Data points close to line, ŵ ≈ w, Small error
  • Case 2: High Noise — Data scattered randomly, Poor estimation, Large error

"Unlucky Dataset" Idea

  • Sometimes noise is unusually high → data becomes misleading.
  • Result: Bad estimate for that dataset.
  • Solution: Evaluate performance on average (expectation), not on one dataset.

Key Insights

What You Cannot Control → Noise variance (σ²)
What You Can Improve → Feature selection, independence, and data quality

One-Line Summary: The goodness of MLE depends on noise in data and quality of features, measured by expected squared error between estimated and true parameters.

Lecture 2: Cross-Validation for Minimizing MSE

Matrix Properties (Essential Basics)

Trace of a Matrix: trace(A) = Σ diagonal elements

Example: A=[[2,1], [0,3]], trace(A) = 2+3 = 5

Trace & Eigenvalues Relation: trace(A) = Σᵢ λᵢ — Sum of eigenvalues = trace

Eigenvalues of Inverse: If eigenvalues of A are λ₁,λ₂,...,λd, then eigenvalues of A⁻¹ are: 1/λ₁, 1/λ₂, …, 1/λd

Eigenvalues of A+λI: λᵢ → (λᵢ+λ) — Adding λI shifts eigenvalues upward

MSE of Maximum Likelihood Estimator

  • Definition: MSE = E[‖Ŵ − W‖²]
  • MLE Case:

E[‖ŴML − W‖²] = σ² Σᵢ₌₁ᵈ 1/λᵢ

Where: λᵢ: eigenvalues of XXᵀ, σ²: noise variance

Problem: Small eigenvalues → very large 1/λᵢ → huge MSE. Weak/correlated features → unstable model

New Estimator (Regularization Idea)

Formula (Ridge Regression):

Ŵnew = (XXᵀ + λI)⁻¹XY

Why It Works:

  • Eigenvalues become: λᵢ+λ
  • So: 1/(λᵢ+λ) < 1/λᵢ

MSE reduces

  • Key Insight:
    • Adding λ prevents very small eigenvalues
    • Stabilizes inversion
    • Reduces overfitting

Theoretical Result: There exists some λ such that: MSEnew < MSEML

Example (Intuition)

  • Without Regularization: Model overfits; Very sensitive to noise
  • With Regularization: Smoother model; Better generalization

Choosing λ (Practical Part)

  • Problem: Best λ is unknown
  • Solution: Cross-Validation

Train–Validation Split

Idea: Split data → 80% train, 20% validation

Process: Train model for different λ values, evaluate error, choose best λ

Example: Total data = 10,000, Train = 8,000, Validate = 2,000

K-Fold Cross-Validation

Idea: Split data into K equal folds

Process:

  • Train on K−1 folds
  • Validate on 1 fold
  • Repeat K times
  • Average errors → choose best λ
Example (3-Fold):
  • Train on (F1, F2) → test on F3
  • Train on (F1, F3) → test on F2
  • Train on (F2, F3) → test on F1

Leave-One-Out Cross-Validation (LOOCV)

  • Idea: K=n (number of data points)
  • Process: Train on n−1 points, test on 1 point
  • Drawback: Very slow for large datasets

Final Intuition (Big Picture)

  • Problem: Small eigenvalues → high MSE
  • Solution: Add λI → stabilize matrix
  • Tradeoff:
    • Too small λ → overfitting
    • Too large λ → underfitting

Lecture 3: Bayesian Modeling for Linear Regression

Core Idea: Bayesian Approach

  • In Bayesian learning, parameters w are random variables, not fixed.
  • We describe them using probability distributions.

Three Main Components

  • Prior (Before seeing data): P(w) — Represents initial belief about w.
  • Likelihood (Data model): P(y ∣X,w) — Probability of data given w.
  • Posterior (After seeing data): P(w∣D) ∝ P(y∣X,w)·P(w) — Updated belief after observing data.

Key Distributions (Linear Regression)

  • Likelihood (Gaussian): y∣x,w ∼ N(wᵀx,σ²) — Often assume σ²=1 for simplicity.
  • Prior (Gaussian): w ∼ N(0,γ²I)
    • Each wᵢ independent
    • Mean = 0
    • Variance = γ²

Conjugate Priors

  • Definition: A prior is conjugate if posterior has the same form.
  • Examples:
    • Bernoulli + Beta → Beta
    • Gaussian + Gaussian → Gaussian

In Linear Regression: Likelihood → Gaussian; Prior → Gaussian; Posterior → Gaussian (easy to compute)

Posterior Formula (Conceptual Form)

P(w∣data) ∝ Πᵢ e−½(yᵢ−wᵀxᵢ)² × e−(1/2γ²)‖w‖²

Interpretation:

  • First term → data fitting (likelihood)
  • Second term → regularization (prior)

MAP Estimation (Most Important)

  • Definition: ŵMAP = argmax P(w | D)

Equivalent Optimization:

minw [ ½ Σᵢ(yᵢ−wᵀxᵢ)² + (1/2γ²)‖w‖² ]

Compared to MLE:

  • MLE → only error term
  • MAP → error + penalty term

MAP Solution (Closed Form)

ŵMAP = (XXᵀ + (1/γ²)I)⁻¹XY

  • Components:
    • XXᵀ: data structure
    • I: identity matrix
    • 1/γ²: regularization strength

Connection to Ridge Regression

Key Result: λ = 1/γ²   →   MAP = Ridge Regression

  • Meaning:
    • Bayesian prior ⇒ regularization
    • Smaller γ² ⇒ stronger penalty

Intuition

  • Without Prior (MLE): Can overfit; Large weights possible
  • With Prior (MAP): Penalizes large weights; Produces smoother model
Example: House Price Prediction
  • Features: Size, Location, Rooms
  • MLE: Fits training data very closely → may overfit
  • MAP: Adds penalty on weights → better generalization

Choosing γ² (or λ)

  • Problem: Best γ² not known beforehand
  • Solution: Cross-Validation — Try different values, pick best based on validation error

Final Intuition (Big Picture)

  • Bayesian View: Prior = belief; Likelihood = data; Posterior = updated belief
  • Why MAP Works Better: Controls model complexity; Avoids overfitting; Stabilizes solution

One-Line Summary: MAP estimation = MLE + prior knowledge, which naturally leads to ridge regression and gives better, more stable models.

Lecture 4: Ridge Regression

Basic Idea

Linear Regression (ML Estimator)

  • Goal: Find weights W that minimize prediction error.
  • Focuses only on fitting the data well.
  • Problem: Can overfit when features are many or redundant.

Ridge Regression (Regularized / MAP Estimator)

  • Adds a penalty term to control model complexity.
  • Also called L2 Regularization.
  • Prevents overfitting by shrinking weights.

ŴR = argminW [ Σᵢ(Wᵀxᵢ−yᵢ)² + λ‖W‖² ]

Role of Regularization

Key Idea: Penalize large weights.

  • Ridge adds penalty: λ‖W‖²
  • Forces weights to be small and stable.

Balancing Two Goals:

  1. Minimize error (fit data well).
  2. Keep weights small (avoid overfitting).

Preference for Simplicity: If multiple solutions give same error → choose smaller weights & simpler model.

Bayesian Perspective

Prior Assumption: W ∼ N(0, γ²)

Hyperparameter Relation: λ ≈ 1/γ²

Effect of λ

λ ValueMeaningEffect
Small λWeak penaltyModel behaves like Linear Regression
Large λStrong penaltyWeights shrink toward zero

λ is chosen using cross-validation.

Handling Redundant Features

Problem: In high-dimensional data, many features may be duplicate or correlated → unstable weights.

Ridge Solution: Pushes unnecessary feature weights towards zero. Avoids assigning importance to redundant features.

Illustrative Example
Features: f₁ = Height, f₂ = Weight, f₃ = 2·Height + 3·Weight
Target: y = 3·Height + 4·Weight

Possible Solutions:
Option A: Weights = (1, 1, 1) → f₁ + f₂ + f₃ = 3·Height + 4·Weight
Option B: Weights = (3, 4, 0) → 3f₁ + 4f₂ = 3·Height + 4·Weight

Which One Does Ridge Choose?
Option A → Larger total weight norm. Option B → Smaller norm.
Ridge selects (3, 4, 0) → simpler, avoids redundancy.

Key Takeaways

Linear Regression: Minimizes only error; Can overfit.

Ridge Regression: Adds penalty term; Controls overfitting; Handles multicollinearity.

λ (Lambda): Controls strength of regularization. Higher λ → simpler model.

Ridge prefers: Small weights; Non-redundant features.

One-Line Summary: Ridge Regression = Linear Regression + L2 penalty. It balances fit vs simplicity, avoids overfitting, and handles redundant features.

Lecture 5: Relation Between Solution of Linear Regression and Ridge Regression

Core Concepts & Formulations

Standard Linear Regression

Objective: Minimize squared prediction error

minw Σ(wᵀxᵢ − yᵢ)²

Maximum Likelihood (ML) View:

  • Linear regression = Maximum Likelihood Estimation
  • Assumption: Noise in data follows Gaussian distribution

Ridge Regression (Regularized Version)

Adds penalty term to avoid overfitting:

minw Σ(wᵀxᵢ − yᵢ)² + λ‖w‖²

Regularization Term: λ‖w‖² = λ Σᵢ₌₁ᵈ wᵢ²

Purpose: Shrinks weights toward zero; Reduces impact of redundant features

Constrained Optimization View

Equivalent Formulation:

minw Σ(wᵀxᵢ − yᵢ)²   subject to ‖w‖² ≤ θ

Key Idea: For every λ, there exists a corresponding θ. Both forms give the same solution.

Geometric Meaning: Constraint ‖w‖² ≤ θ forms a circle (2D) centered at origin.

Geometry of Loss (Contours)

ML Solution: ŵML → gives minimum loss.

Constant Loss Sets: Points with same loss form contours.

Contour Equation:

(w − ŵML)ᵀ XᵀX (w − ŵML) = C′

Shape of Contours:

  • If XᵀX = I → Circle
  • General case → Ellipse

Loss surface is usually elliptical.

How Ridge Finds the Solution

Key Idea: Combine elliptical loss contours with circular constraint region.

Process:

  1. ML solution lies outside constraint circle.
  2. Move to smaller weights (inside circle).
  3. Find first point of intersection.

Final Answer: Ridge solution = Point where smallest ellipse touches the circle.

Result: ŵRidge has: Smaller magnitude; Less overfitting. Called shrinkage of weights.

Example (Intuition)
Suppose:
  • Features: Height, Weight, Redundant Feature
  • ML gives large weights (overfitting).
Ridge Effect: Shrinks all weights. Redundant feature weight becomes very small, but usually not exactly zero.

Key Insights

Shrinkage vs Sparsity:

  • Ridge shrinks weights.
  • Does NOT make them exactly zero.

Feature Selection Issue: Since weights ≠ 0 → Hard to remove useless features.

Why Explore Other Methods? Shape of constraint matters:

RegularizerShapeEffect
Ridge (L2)CircleSmooth shrinkage
Lasso (L1)DiamondCan force zeros

Leads to Lasso Regression for feature selection.

Final Quick Revision Points

Linear Regression: Minimizes only error & Sensitive to overfitting

Ridge Regression: Adds L2 penalty; Controls overfitting; Handles redundancy

Geometry: Loss → Ellipses; Constraint → Circle; Solution → Intersection point

Limitation: No exact feature selection

One-Line Summary: Ridge Regression = Linear Regression + L2 penalty. It finds the intersection of elliptical loss contours and a circular constraint, producing smaller, more stable weights.

Lecture 6: Relation Between Solution of Linear Regression and Lasso Regression

Key Concepts

What is LASSO? LASSO = Least Absolute Shrinkage and Selection Operator

  • A regression technique that: Minimizes error; Shrinks weights; Performs feature selection

Meaning of the Name:

  • Least → Minimize loss
  • Absolute → Uses absolute values (L1 norm)
  • Shrinkage → Reduces weight values
  • Selection → Sets some weights exactly to zero

Sparsity (Most Important Idea)

  • LASSO produces a sparse solution → many weights become: wᵢ = 0

Helps identify: Important features ✅; Redundant features ❌

Mathematical Formulation

L1 Norm Definition: ‖w‖₁ = Σᵢ₌₁ᵈ |wᵢ|

LASSO Objective Function:

minw Σᵢ(wᵀxᵢ−yᵢ)² + λ‖w‖₁

Two parts: Error term → Fit data; L1 penalty → Force sparsity

Constrained Form:

minw Σᵢ(wᵀxᵢ−yᵢ)²   subject to ‖w‖₁ ≤ θ

Geometric Intuition (Very Important)

Constraint Regions:

  • Ridge (L2 Regularization): Shape: Circle (2D); Smooth boundary; Rarely produces zero weights
  • LASSO (L1 Regularization): Shape: Diamond; Sharp corners on axes

Why LASSO Gives Sparsity?

Key Idea: Loss function forms elliptical contours expanding from ML solution.

What Happens:

  • Ellipses grow outward.
  • First intersection with constraint region occurs.

In LASSO: Diamond corners lie on axes. Intersection often happens at corners.

Result: Some weights become wᵢ = 0. This is automatic feature selection.

Example (Easy Understanding)
Suppose: Features: f₁ = Height, f₂ = Weight, f₃ = Redundant feature

LASSO Behavior:
  • Keeps important features: w₁, w₂ ≠ 0
  • Removes useless feature: w₃ = 0
Model becomes simpler + interpretable.

High-Dimensional Advantage

When features are very large (1000+): Many features are redundant. LASSO removes unnecessary ones, keeps only useful signals.

Compared to Ridge

FeatureRidge (L2)LASSO (L1)
Weight shrinkageYesYes
Exact zero weights❌ No✅ Yes
Feature selection❌ No✅ Yes
GeometryCircleDiamond

Final Quick Revision Points

LASSO uses L1 regularization; Produces sparse solutions; Performs automatic feature selection; Works well for high-dimensional data

Key idea: Diamond shape → corner intersection → zero weights

One-Line Summary: LASSO Regression = Linear Regression + L1 penalty. It produces sparse solutions by setting some weights exactly to zero, enabling automatic feature selection.

Lecture 7: Characteristics of Lasso Regression

Ridge vs LASSO – Core Comparison

FeatureRidge Regression (L2)LASSO Regression (L1)
Regularizationλ‖w‖²λ‖w‖₁
Closed-form solution✅ Yes❌ No
Sparsity❌ No (weights ≠ 0)✅ Yes (some weights = 0)
Feature Selection❌ No✅ Yes
DifferentiabilitySmoothNot differentiable at 0
ComputationDirect solutionIterative methods

Closed-Form Solution

Ridge Regression Solution: w = (XᵀX + λI)⁻¹Xᵀy — Fast and direct (for small datasets).

LASSO Limitation:

  • No closed-form solution.
  • L1 norm has sharp corner at 0 → not differentiable.

LASSO Optimization Problem

minw Σ(wᵀxᵢ − yᵢ)² + λ‖w‖₁

Convexity:

  • Squared loss → convex
  • L1 norm → convex
    • Sum of convex functions → convex problem
    • Guarantees global minimum.

Optimization Methods for LASSO

1. Subgradient Descent

  • Works even when function is not differentiable.
  • General-purpose method.

2. IRLS (Iterative Reweighted Least Squares)

  • Special method for regression problems.
  • Uses repeated weighted least squares.

Concept of Subgradients

Definition: A vector g is a subgradient if: f(z) ≥ f(x) + gᵀ(z−x)

Intuition: Subgradient = generalization of gradient. Works even when function has sharp corners.

Key Idea: Gradient = single direction. Subgradient = multiple possible directions.

Important Example (Absolute Value)

Function: f(x) = |x|

At x=0: Not differentiable. Subgradient can be: g ∈ [−1,1] — Infinite possible directions!

In Optimization: Pick any subgradient. Move in negative direction. Eventually reach minimum.

Why LASSO Works (Revisited)

  • L1 penalty has sharp corners.
  • Optimization often hits axes → weights become zero.
    • Leads to feature selection.

Other Regularization Methods

Elastic Net: Combination of L1 + L2: λ₁‖w‖₁ + λ₂‖w‖₂

Benefits: Sparsity (L1); Stability (L2).

Domain-Specific Regularizers: Custom penalties based on feature groups or domain knowledge. Example: Group features selected together.

Evolution of Regression Models

Step-by-Step Progression:

  1. Least Squares → Minimize squared error, closed-form solution.
  2. Maximum Likelihood (ML) → Probabilistic view, assumes Gaussian noise.
  3. Ridge Regression (L2) → Adds Gaussian prior, shrinks weights.
  4. LASSO (L1) → Introduces sparsity, performs feature selection.

Final Quick Revision Points 🚀

Ridge: Closed-form solution; No feature selection.

LASSO: No closed-form; Uses subgradients; Produces sparse model.

Subgradient: Works for non-differentiable functions; Key to solving LASSO.

Convexity: Ensures global optimum.

One-Line Summary: Ridge shrinks weights smoothly (no zeros), while LASSO forces sparsity by setting some weights exactly to zero, enabling automatic feature selection.
