Week 2: Short Notes

Lecture 1: Issues With PCA

Real-World Application: Eigenfaces

  • Dataset Example: Yale Face Dataset (~20,000 grayscale images, same size, front-facing).
  • Feature Engineering:
    • Image of size m × n.
    • Vectorization: reshape into (mn × 1).
    • Example: 32,000 pixels → 32,000-dimensional vector.

Principal Components (Eigenfaces)

  • Each eigenvector: 32,000-dimensional.
  • Reshape back → image (Eigenface).
  • Key Insights:
    • PC1 (First Principal Component): captures maximum variance, represents "average face" or faceness.
    • Other PCs: capture finer details (nose shape, mouth variations, lighting differences).

Image Reconstruction & Compression

  • Process:
    • Take test image vector x.
    • Project onto top k principal components.
    • Reconstruct using linear combination.
  • Compression Power:
    • Original: 32,000 pixels.
    • Reduced: 200–400 components.
    • Approx. 100× compression.
  • Reconstruction Quality:
    • 200–400 PCs → good for recognition.
    • ~1600 PCs → near-perfect reconstruction.
  • Important Insight:
    • PCA is data-specific.
    • Example: Human face PCs cannot reconstruct dog faces well.

Major Issues with PCA

Issue 1: High Time Complexity

  • Formula: O(d³), where d = number of features.
  • Comes from eigen decomposition of covariance matrix.
  • For large d (e.g., 32,000), computation is slow/impractical.

Issue 2: Cannot Handle Non-Linear Data

  • PCA assumes data lies in a linear subspace.
  • → non-linear relationship.

Manifold Problem

  • Real-world data often lies on curved surfaces (manifolds).
  • PCA fits a flat plane → high reconstruction error.

Key Concepts & Formulae

  • Vectorization: Image (m × n) → (mn × 1).
  • Covariance Matrix: C ∈ Rd×d.
  • Time Complexity: O(d³).
  • PCA Assumption: Data lies in a low-dimensional linear subspace.
  • Pre-processing Use:
    • PCA applied before classification/regression.
    • Reduces noise and dimensionality.
Final Summary: PCA finds the most important directions of variation and uses them to compress, reconstruct, and understand data — but only if the structure is linear.

Lecture 2: Time Complexity Issue With PCA

Core Concepts & Definitions

  • Data Representation
    • Data points: x₁, x₂, …, xn ∈ Rd
    • Data matrix: X = [x₁ x₂ … xn] ∈ Rd×n

Covariance Matrix: C = (1/n) Σᵢ xᵢxᵢᵀ = (1/n) XXᵀ   (Size: d×d)

Eigen-Decomposition: Cwk = λkwk

wk: eigenvector (principal direction), λk: eigenvalue (variance captured)

  • Complexity Issue: Eigen-decomposition requires O(d³), expensive when d is large.

Key Insight: Eigenvectors as Linear Combination

  • Each principal component is a linear combination of data points:

wk = Xαk = Σᵢ αkixᵢ, where αk ∈ Rⁿ are weights.

  • Instead of solving in d-dimensional space, solve in n-dimensional space.

Efficient Reformulation (Smaller Matrix)

  • Define Gram Matrix: K = XᵀX ∈ Rn×n
  • Derived Eigen Equation: Kαk = (nλk)αk
  • Key Fact: Non-zero eigenvalues of XXᵀ and XᵀX are the same. Maximum non-zero eigenvalues: min(d,n).

Normalization & Constraints

  • Unit Length Constraint: wkᵀwk = 1
  • Constraint on Coefficients: αkᵀKαk = 1
  • Relation with Eigenvector of K: If βkᵀβk = 1, then αk = 1/√(nλk) βk

Efficient PCA Algorithm (When d ≫ n)

  1. Compute Gram Matrix: K = XᵀX, Kij = xᵢᵀxⱼ
  2. Eigen-decomposition of K: Kβk = (nλk)βk — Complexity: O(n³).
  3. Compute Coefficients: αk = 1/√(nλk) βk
  4. Recover Principal Components: wk = Xαk

Example

  • Features: d = 10,000
  • Data points: n = 100
  • Standard PCA: Solve 10,000 × 10,000 matrix → very slow.
  • Efficient PCA: Solve 100 × 100 matrix → huge speedup.

Key Insight: Pairwise Similarities

  • PCA can be solved using only dot products: xᵢᵀxⱼ
  • No need to explicitly use full feature vectors.
  • Enables advanced methods like Kernel PCA for non-linear data.
Final Summary: When features are huge, PCA can be made efficient by working in the data space (n) instead of the feature space (d), using pairwise similarities.

Lecture 3: Feature Transformation

Overview of PCA Issues

High-Dimensional Features

  • Covariance matrix size: d × d.
  • Eigen-decomposition cost: O(d³).
  • Problem: Too slow when d ≫ n.
  • Solution: Use smaller matrix XᵀX ∈ Rn×n and convert results back.

Nonlinear Relationships

  • PCA assumes data lies in a linear subspace.
  • Fails when data follows curved/nonlinear patterns.

Failure Example: Data on a Circle

  • Setup: Data lies on a circle centered at (a,b): (f₁−a)² + (f₂−b)² = r²
  • What PCA Does:
    • Centers data at (0,0).
    • Finds directions of maximum variance.
  • Problem:
    • Data uniformly spread.
    • No dominant direction.
    • w₁ captures ~50% variance, w₂ captures ~50% variance.
    • PCA cannot reduce dimension.
    • Wrong conclusion: Data is 2D, but actually lies on 1D circle.

Solution: Feature Mapping

  • Idea: Convert nonlinear data → linear in higher dimension.

Original Nonlinear Equation: f₁² + f₂² − 2af₁ − 2bf₂ + (a² + b² − r²) = 0

Feature Mapping Example (2D → 6D): φ(f₁, f₂) = [1, f₁², f₂², f₁f₂, f₁, f₂]ᵀ

Linear Representation in High-D Space: u = [a² + b² − r², 1, 1, 0, −2a, −2b]ᵀ, φ(x)ᵀu = 0

  • Meaning: Data lies in a linear subspace in 6D → PCA can work.

Method: Feature Mapping + PCA

  1. Map data x ∈ Rd → φ(x) ∈ RD.
  2. Apply PCA in higher dimension.
  3. Handle large D using similarity trick: φ(xᵢ)ᵀφ(xⱼ) instead of explicit computation.

Computational Challenge (Curse of Dimensionality)

  • Feature Explosion: If original features = d, degree = p, Total dimensions = Σᵢ₌₀ᵖ (d choose i) ≈ O(dᵖ)
  • Examples: Cubic case (p = 3, d = 4):
    • Constant: 1, Linear: 4, Quadratic: 6, Cubic: 4 → Total = 15.
    • Extreme case (d = 10, p = 15): 10¹⁵ features → impossible to compute.
Final Summary:
  • PCA fails for nonlinear data because it only finds straight-line structure.
  • By mapping data to higher dimensions: nonlinear → linear, PCA becomes effective.
  • But high dimension leads to computation explosion.

One-Line Summary: Make data linear by going high-dimensional, but avoid explicit computation using similarity tricks.

Lecture 4: Kernel Functions

Core Problem: Computational Hurdle

  • Issue: In nonlinear PCA, we map x ∈ Rd → φ(x) ∈ RD. But D can be very large or infinite.
  • Key Observation: PCA does not need φ(x) directly, only φ(X)ᵀφ(X), i.e., pairwise dot products: φ(xᵢ)ᵀφ(xⱼ)
  • Main Question: Can we compute this without explicitly computing φ(x)?

Kernel Function (Core Idea)

  • Definition: A kernel function computes dot product in high-dimensional space directly:

k(x, x′) = φ(x)ᵀφ(x′)

  • Input: low-dimensional vectors. Output: equivalent dot product in high dimension.
  • Intuition: Kernel = shortcut to avoid explicit feature mapping.

Important Kernel Examples

(A) Polynomial Kernel

Formula: k(x, x′) = (xᵀx′ + 1)p, p ≥ 1

  • Example (p = 2):
    • Input: x = [f₁, f₂], x′ = [g₁, g₂]
    • Kernel: (xᵀx′ + 1)²
  • Equivalent Mapping (Hidden): φ(x) = [f₁², f₂², 1, 2f₁f₂, 2f₁, 2f₂]ᵀ — Maps 2D → 6D.
  • Key Insight: Compute 6D dot product using only 2D inputs.

(B) RBF / Gaussian Kernel

Formula: k(x, x′) = exp( −‖x − x′‖² / 2σ² )

  • Key Properties:
    • Maps to infinite-dimensional space.
    • Very powerful for nonlinear data.
  • Intuition: Acts like similarity measure:
    • Close points → high value.
    • Far points → near zero.

Valid Kernel Condition (Mercer's Theorem)

  • Condition 1: Symmetry: k(x,x′) = k(x′,x) — Because dot product is symmetric.
  • Condition 2: Positive Semi-Definite (PSD): For matrix Kij = k(xᵢ, xⱼ), must satisfy: all eigenvalues ≥ 0.
  • Reason:
    • K behaves like covariance matrix.
    • Eigenvalues = variance.
    • Variance is always non-negative.

Big Picture: Kernel Trick

  • Idea: Instead of explicit mapping φ(x), use kernel function: k(x, x′) = φ(x)ᵀφ(x′)
  • Benefit:
    • Avoid high/infinite dimensional computation.
    • Handle nonlinear data efficiently.
Final Summary:
  • Kernel Trick = "Compute in high dimension without going there."

One-Line Summary: Kernels allow PCA (and ML models) to handle nonlinear data efficiently by replacing dot products with smart functions.

Lecture 5: Kernel PCA

Objective & Overview

  • Goal: Apply PCA on nonlinear data.
  • Standard PCA: Works only for linear relationships.
  • Kernel PCA: Uses kernel trick, maps data into high-dimensional space, finds linear structure there.

Core Idea: Nonlinear in original space ⇒ Linear in feature space.

Key Inputs

  • Dataset: {x₁, x₂, …, xn}, xi ∈ Rd
  • Kernel Function: K(xᵢ, xⱼ) = φ(xᵢ)ᵀφ(xⱼ) — Computes dot product in high dimension. Must satisfy: symmetry and positive semi-definiteness.

Kernel PCA Algorithm

  • Step 1: Compute Kernel Matrix: Kij = K(xᵢ, xⱼ) — Size: n × n. Contains pairwise similarities.
  • Step 2: Eigen Decomposition: Kβk = (nλk)βk
    • βk: eigenvectors, λk: eigenvalues

Normalization: αk = 1/√(nλk) βk

  • Step 3: Projection (Dimensionality Reduction): For data point xi, k-th component:

Projection = Σⱼ αkj K(xᵢ, xⱼ)

Result: xi ∈ Rd → Rl, where l = number of principal components.

Centering the Kernel

  • Why Needed: PCA requires mean = 0.
  • Problem: Cannot compute φ(x) directly.
  • Solution: Center kernel matrix instead. Kc = centered version of K

Idea: Adjust row means, column means, and overall mean to mimic (φ(xᵢ) − μ)ᵀ(φ(xⱼ) − μ)

Important Concepts

  • Kernel Trick: Replace xᵢᵀxⱼ → K(xᵢ, xⱼ). Avoid explicit mapping.
  • Dimensionality Insight: Can increase dimension first, then reduce.
    • Examples: 2D → 100D → 10D
    • 100D → 5D
  • Use in Practice: Feature preprocessing before classification or regression.

Examples

Example 1 (Reduction): Input: 100D data → Output: Top 5 nonlinear components. Captures most important patterns.

Example 2 (Nonlinear Data): Input: 2D data with nonlinear structure. Map → 100D feature space → Reduce → 10D. Structure becomes linear and separable.

Final Summary: Kernel PCA = "Apply PCA in a hidden high-dimensional space using kernels."

One-Line Summary: Kernel PCA uses kernels to capture nonlinear patterns by performing PCA in an implicit high-dimensional space without computing it directly.
