Week 7: Short Notes

Lecture 1: Introduction to Binary Classification

Core Concepts

Supervised Learning

  • Input: Features (x)
  • Output: Label (y)
  • Goal: Learn a function h(x) that maps input features to one of two classes.
  • Labels: Common formats: {0,1} or {−1,1} (choice is for convenience).

Linear Classifier

  • Uses a weight vector (w).
  • Decision rule: Based on wᵀx.
  • Decision boundary: wᵀx = 0

Splits space into two regions: one side → Class 1, other side → Class 0 / -1.

NP-Hard Problem

  • Minimizing classification error (0–1 loss) is NP-hard.
  • Reason: Loss is discrete → non-continuous → cannot use derivatives → hard optimization.

Key Formulae

0–1 Loss Function

L(h) = (1/n) Σᵢ I(h(xᵢ) ≠ yᵢ)

Where I(condition)=1 if TRUE, else 0.

Linear Hypothesis Function: hw(x) = sign(wᵀx)

Sign Function

sign(z) = 1 if z > 0;   0 or −1 if z ≤ 0

Examples

Email Spam Detection
  • Input: Email features (words, links, etc.)
  • Output: 1 → Spam, 0 → Not Spam

Geometric View (2D Case)

  • Line: wᵀx = 0
  • Points:
    • One side → Positive class
    • Other side → Negative class

Classification vs Regression

FeatureClassificationRegression
OutputDiscrete (class labels)Continuous values
Loss0–1 Loss (discrete)Squared Error (continuous)
GoalCorrect classAccurate value

Why Linear Regression is NOT Good for Classification

  • Sensitive to Outliers: Squared error → large errors dominate.
  • Tilt Problem: Far-away points shift decision boundary → misclassification.
  • Wrong Objective: Regression cares about exact value, classification cares only about sign.
Final Summary (Memory Trick):
  • Binary classification → 2 classes.
  • Use sign(wᵀx) for prediction.
  • Boundary → wᵀx=0.
  • Loss → 0–1 (count mistakes).
  • Hard to optimize → NP-hard.
  • Regression ❌ for classification (tilt + outliers).

Lecture 2: K-Nearest Neighbours

Core Concept (Nearest Neighbor)

  • Idea: Classify a new point based on similar existing data.
  • 1-Nearest Neighbor (1-NN):
    • Find closest training point x*.
    • Assign its label to the test point.
    • Rule: ytest = y*
  • Problem: Highly sensitive to outliers. One wrong neighbor → wrong prediction.

K-NN Algorithm (Improved Version)

Instead of 1 neighbor → use k neighbors.

Steps:

  1. Compute distance from test point to all training points.
  2. Select k nearest neighbors.
  3. Take majority vote.

Prediction Rule: ytest = majority label among k neighbors

Reduces effect of outliers.

Distance Function

  • Most common: Euclidean Distance d(x,y) = Σ(xᵢ−yᵢ)²
  • Other options: Manhattan distance; Cosine similarity.

Decision Boundary

  • Small k (e.g., k=1): Complex, irregular boundary. Sensitive to noise.
  • Large k (e.g., k=n): Very smooth boundary. Ignores local structure.
  • Optimal k: Balance between noise handling and data structure.

Choosing Best k

Use Cross-Validation:

  1. Try multiple k values.
  2. Evaluate accuracy on validation set.
  3. Choose best k.
Example
Dataset: Class +1 (Red), Class −1 (Blue).
Test point near 3 red, 2 blue neighbors.
Prediction: y = +1
Because majority = red.

Advantages

  • Simple & intuitive.
  • No training phase.
  • Works well for small datasets.

Major Issues of K-NN

  1. Distance Problem
    • Not all features equally important.
    • Scaling issues.
  2. Computational Cost
    • For each test point: compute distance to all training points, sort to find k nearest.
    • Very slow for large datasets.
  3. No Learning (Lazy Learning)
    • K-NN does not build a model.
    • Stores entire dataset.
    • Problems: high memory usage, no abstraction of data.

Key Insight

K-NN is called a Lazy Learning Algorithm. Because: No training; Learning happens during prediction.

Quick Revision Summary:
  • 1-NN: Uses 1 neighbor → sensitive to noise.
  • K-NN: Uses k neighbors → majority vote.
  • Key parameter: k (hyperparameter).
  • Distance: Usually Euclidean.
  • Trade-off: Small k → overfitting; Large k → underfitting.
  • Weakness: Slow + no model.

Lecture 3: Introduction to Decision Trees

Introduction

  • Decision Tree is a model-based learning algorithm.
  • Unlike K-NN: Learns a model (tree) from data; No need to store full dataset after training.
  • Benefits: Fast prediction; Easy to understand; Foundation for advanced models (Random Forest, Boosting).

Structure of a Decision Tree

Components:

  • Root Node → Starting point.
  • Internal Nodes → Questions (yes/no).
  • Leaf Nodes → Final prediction.

Key Idea: Each path = sequence of decisions. Each leaf = predicted label.

Prediction Mechanism

For a test point xtest:

  1. Start at root.
  2. Answer questions: YES → go left; NO → go right.
  3. Reach a leaf.

Output: Label at leaf. Advantage: Very fast prediction (O(depth)).

Nature of Questions

Questions are of type: Feature ≤ Threshold?

Example: Is height ≤ 180 cm? Is age ≤ 25?

Effect: Splits data into subsets Dyes and Dno.

Building the Tree (Greedy Approach)

At each node: Choose best question. Goal: Make subsets as pure as possible.

Concept of Purity

  • Pure node → all labels same.
  • Impure node → mixed labels.

Entropy (Impurity Measure)

H(p) = −p log₂p − (1−p)log₂(1−p)

Properties:

  • H=0 → perfectly pure.
  • H=1 → maximum impurity (50-50 split).
  • Symmetric: 25%-75% = same entropy as 75%-25%.

Information Gain

Measures how good a split is.

IG = H(D) − [γH(Dyes) + (1−γ)H(Dno)]

Where: γ = |Dyes| / |D|

Idea: High gain = better split. Choose question with maximum IG.

Example

HeightLabel
170+1
180+1
160-1
150-1

Possible Question: Is height ≤ 165?

  • YES → {160,150} → all -1 (pure).
  • NO → {170,180} → all +1 (pure).

Entropy becomes 0 after split. Information Gain = maximum.

Key Insights

  • Decision tree builds step-by-step (greedy).
  • At each step: choose best feature + threshold.
  • Stops when: Data becomes pure; Or max depth reached.

Advantages

  • Easy to understand.
  • Fast prediction.
  • Works with non-linear data.
  • No need for feature scaling.

Limitations

  • Can overfit (very deep tree).
  • Sensitive to small data changes.
  • Greedy approach → not globally optimal.
Quick Revision Summary:
  • Tree = questions + answers → prediction.
  • Metric: Entropy → impurity.
  • Information Gain → split quality.
  • Goal: Maximize IG, minimize impurity.
  • Prediction: Follow path root → leaf.

Lecture 4: Decision Tree Algorithm

Decision Tree Algorithm Steps

Step 1: Choosing Questions

  • For each feature, create questions like: fk ≤ θk?
  • For continuous features: convert into intervals (discretization).

Step 2: Select Best Split

Choose question with maximum Information Gain:

IG = H(D) − [γH(Dyes) + (1−γ)H(Dno)]

Step 3: Recursive Partitioning

  • Split dataset into Dyes and Dno.
  • Repeat process on each subset.

Step 4: Pruning Redundant Questions

  • Avoid useless splits.
  • Example: If already fk ≤ 5, then asking fk ≤ 10 is redundant.

Stopping Criteria & Hyperparameters

Trees grow greedily but must stop.

(A) Node Purity

  • Stop if node is almost pure.
  • Example: 90% same label → stop.

(B) Maximum Depth

  • Limit number of levels in tree.
  • Depth = number of questions from root to leaf.

(C) Cross-Validation

  • Try different depths.
  • Choose best accuracy.

Impurity Measures

(A) Entropy: H(p) = −p log₂p − (1−p)log₂(1−p)

(B) Gini Index: G(p) = 1−p² − (1−p)²

Properties:

  • = 0 → pure node.
  • Maximum at p=0.5.
  • Symmetric.

Decision Boundaries

  • Each split = axis-parallel line.
  • Result: Feature space divided into rectangles.

Overfitting vs Generalization

  • Overfitting: Very deep tree, memorizes noise, creates tiny regions.
  • Generalization: Moderate depth, captures real pattern.

Analogy: Depth in trees ≈ k in K-NN.

Key Advantages

  • Interpretability: Easy to explain decisions.
  • Human-Readable: Small trees (~10 levels) are understandable.
  • Model-Based: No need to store dataset after training.

Limitations

  • Overfitting risk.
  • Sensitive to small changes.
  • Greedy (not globally optimal).
Quick Revision Summary:
  • Build tree: choose best split (max IG).
  • Recursively divide data.
  • Stop using: purity, max depth.
  • Impurity: Entropy / Gini.
  • Boundary: Rectangular regions.
  • Trade-off: Depth vs generalization.

Lecture 5: Generative and Discriminative Models

Core Problem of Learning

  • Main Challenge: Model must perform well on unseen (test) data, not just training data.
  • Problem with Memorization: Memorizing training data = no generalization. Fails on new inputs.
  • Key Requirement: There must be a relationship between train & test data.
  • Example: If trained on student marks → should work on new students.

Fundamental Distribution Assumption

  • Key Idea: Data comes from an unknown distribution D.
  • Representation: (x,y) ∼ D
  • x → features, y → label.
  • Assumption: Training data and test data are drawn from the same distribution.
  • Objective: Learn patterns of D to predict correctly.

Two Types of Modeling

A. Generative Models

  • Definition: Model joint probability P(x,y).
  • Idea: Learn how data is generated.

Process:

P(y|x) = P(x,y) / P(x)

  • Focus: Understand data generation mechanism.
  • Examples: Naive Bayes, Gaussian Naive Bayes.
  • Analogy: If you know a language, you can generate sentences and identify the language easily.

B. Discriminative Models

  • Definition: Model conditional probability P(y∣x).
  • Idea: Directly learn decision boundary.
  • Focus: Only care about classification.
  • Examples: K-Nearest Neighbors (KNN), Decision Trees, Logistic Regression.
  • Analogy: You can't speak a language, but can still recognize it.

Key Differences

AspectGenerativeDiscriminative
Probability ModeledP(x, y) (joint probability)P(y | x) (conditional probability)
FocusLearn how data is generatedDirectly learn decision boundary / classification
ComplexityUsually higher, requires modeling feature distributionUsually simpler, focuses only on labels
RiskWrong assumptions can hurt performanceSafer, fewer assumptions about features

Choosing Between Models

  • Use Generative When:
    • You know how data is generated.
    • Strong domain knowledge.
  • Use Discriminative When:
    • Feature generation is unknown.
    • Most real-world cases.
  • Important: Wrong generative assumptions → bad performance.

Examples in Practice

  • K-Nearest Neighbors (KNN): Discriminative, predicts using neighbors.
  • Decision Trees: Discriminative, uses feature-based questions.
  • Key Insight: These models don't model P(x).

Trivial Probabilities

  • In many discriminative models, output is deterministic: 1 (certain) or 0 (not possible).
  • Example: Decision Tree leaf → fixed label. KNN → majority class.
Quick Revision Summary:
  • Goal: Generalize from train → test.
  • Assumption: Data comes from same distribution D.
  • Models: Generative → learn P(x,y). Discriminative → learn P(y∣x).
  • Real-world: Discriminative models are more common.

One-Line Intuition: Generative = "How data is created." Discriminative = "How to classify data."
