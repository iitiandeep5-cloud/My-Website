Week 3: Short Notes

Lecture 1: Introduction to Clustering

What is Clustering?

  • Clustering is an unsupervised learning technique.
  • It groups data points into K clusters based on similarity.
  • Unlike PCA (which reduces dimensions), clustering finds natural groupings in data.

Key idea: Even if data lies in a linear space, it may still form distinct groups (clusters).

Core Concepts

Dataset: Given points: x₁, x₂, …, xn ∈ RD

Cluster Assignment (Indicator Variable)

Each point is assigned a cluster: zi ∈ {1, 2, …, K}

Meaning: zi = k ⟹ xi belongs to cluster k

Homogeneity (Goal)

Points in the same cluster should be:

  • Similar
  • Close to each other

Measured using distance from cluster center.

Key Formulae

1. Cluster Mean (Centroid)

The center of cluster k:

μk = Σᵢ xᵢ·I(zi=k) / Σᵢ I(zi=k)

Where I(zi = k) = 1 if point belongs to cluster k, else 0. Denominator = number of points in cluster k.

Simply: Mean of all points in that cluster.

2. Objective Function (Clustering Cost)

Measure of clustering quality:

Σᵢ ‖xᵢ − μzᵢ‖²

Goal: Minimize total squared distance between each point and its cluster center. Lower value = better clustering.

Computational Complexity

Why Clustering is Hard? Total possible assignments: Kn

Example: For K = 2, n = 1000: 21000 (extremely large!)

Conclusion: Finding exact best clustering = NP-HARD. So we use heuristic algorithms (like K-means).

Examples

Example 1: Number of Partitions
Points: x₁, x₂, x₃, x₄, x₅
Clusters: K=3
Total ways: 3⁵ = 243

Sample grouping:
  • Cluster 1: x₁, x₂, x₅
  • Cluster 2: x₃
  • Cluster 3: x₄
Example 2: Mean & Cost Calculation
Given:
  • Cluster 1: x₁, x₃, x₄
  • Cluster 2: x₂, x₅
Means: μ₁ = (x₁+x₃+x₄)/3,   μ₂ = (x₂+x₅)/2
Cost: ‖x₁−μ₁‖² + ‖x₂−μ₂‖² + ‖x₃−μ₁‖² + ‖x₄−μ₁‖² + ‖x₅−μ₂‖²
Quick Summary:
  • Clustering = grouping similar data points
  • Each point assigned using zi
  • Cluster center = mean of assigned points
  • Objective = minimize squared distance
  • Problem is NP-HARD → use approximations

Lecture 2: K-means Clustering (Lloyd's Algorithm)

Core Concepts

K-Means Problem vs Algorithm

  • K-Means Problem: Find the best way to divide data into K clusters.
  • Lloyd's Algorithm: A heuristic method used to solve the problem (not guaranteed optimal).

Important: Often called the K-means algorithm, but technically it is Lloyd's Algorithm.

Complexity

  • Finding the optimal clustering = NP-HARD
  • Lloyd's Algorithm:
    • Simple
    • Fast in practice
    • May give local optimum (not always best solution)

Iterative Nature: The algorithm runs repeatedly until convergence.

Convergence condition:

  • No point changes its cluster
  • Every point is closest to its current cluster center

Algorithm Steps

Step 0: Initialization: Assign each point a random cluster label: zi(0) ∈ {1, 2, …, K}

Step 1: Compute Cluster Means: For each cluster k, compute its centroid:

μk(t) = Σᵢ xᵢ·I(zi(t)=k) / Σᵢ I(zi(t)=k)

Meaning: Take average of all points in cluster k.

Step 2: Reassignment Step: Reassign each point to the nearest cluster center:

zi(t+1) = argmink ‖xᵢ − μk(t)‖²

Meaning: Choose cluster with minimum squared distance.

Tie-Breaking Rule

If distances are equal:

  • Do not move the point
  • Stay in current cluster

Movement only happens if another cluster is strictly closer.

Key Properties

Convergence

  • Algorithm is guaranteed to converge.
  • Reason: Cost (distance) always decreases or stays the same.
  • Finite number of partitions.

Initialization Matters

  • Different starting points → different results.
  • Can affect: Speed, Final clusters
  • Common issue: local minima.

Choosing K

  • Number of clusters K is not given in unsupervised learning.
  • Must be chosen using:
    • Domain knowledge
    • Methods like the Elbow method (conceptually).

Nature of Clusters

Works best when clusters are: Spherical, Well-separated.
Struggles with: Irregular shapes, Different densities.

Example (Movement of Points)
Given: A point currently in Cluster A
  • Case 1: Distance to A = 5, Distance to B = 6 → Stays in Cluster A
  • Case 2: Distance to A = 5, Distance to B = 5 → Stays in Cluster A (tie-breaking rule)
  • Case 3: Distance to A = 5, Distance to B = 3 → Moves to Cluster B
Quick Summary:
  • K-Means = partition data into K clusters
  • Uses Lloyd's Algorithm (iterative)
  • Steps: Initialize clusters → Compute means → Reassign points
  • Objective: minimize distance to cluster centers
  • Guaranteed to converge, but: Not always optimal; Sensitive to initialization and choice of K

Lecture 3: Convergence of K-means Algorithm

Overview

What is K-Means? An unsupervised learning algorithm used to divide data into K clusters.

How it Works

  1. Start with an initial random partition
  2. Repeat: Compute cluster means → Reassign points to nearest cluster
  3. Stop when no point changes cluster

Movement Rule: A point shifts cluster only if distance to another cluster mean is strictly smaller. Uses squared Euclidean distance.

Termination Condition

Algorithm stops when:

  • No reassignment occurs
  • All points are "happy" with their cluster

Key Mathematical Concept (Fact 1)

Problem: Given points: x₁, x₂, …, xl ∈ Rd

Find vector v minimizing:

minv Σᵢ ‖xᵢ − v‖²

Result: The optimal v* is:

v* = (1/l) Σᵢ xᵢ

Intuition: The mean is the best representative point. It minimizes total squared distance.

Why K-Means Converges

Objective Function: Total clustering cost:

F = Σᵢ ‖xᵢ − μzᵢ‖²

Goal: Minimize F.

A. Two-Step Reduction

  1. Reassignment Decreases Cost: When a point moves, the distance to the new (closer) cluster is smaller than to the old one.
  2. Mean Update Further Reduces Cost: After recomputing means, cost decreases further (from Fact 1: mean minimizes distance).

Final Insight: Each iteration: Cost strictly decreases OR stays same. Never increases.

B. Finiteness Argument

Total possible partitions: Kn (finite)

Since:

  • Cost decreases every time
  • Same partition never repeats

Algorithm must stop (converge).

Key Concepts & Caveats

Local Minima: Algorithm may stop at a local optimum, not the global best. Reason: No single point wants to move.

Speed: Usually very fast in practice. Worst-case: can take many iterations (rare).

Choosing K: Two Cases

  • Predefined K: Example – grading system
  • Unknown K: Need to discover patterns, use heuristics (e.g., elbow idea)

Examples

Example 1: Educational Grading
Teacher groups students into: S, A, B, C, D
Here: K = 5. Algorithm clusters students based on marks.
Example 2: Pattern Discovery
Given only data points, no predefined K.
Goal: Find natural grouping structure. Try different K values to see patterns.
Quick Summary:
  • K-Means = iterative clustering algorithm
  • Uses: Mean as cluster center; Squared distance
  • Convergence reason: Cost decreases every step; Finite number of partitions
  • Guarantees: Converges; Not always optimal

Lecture 4: Nature of Clusters Produced by K-means

Core Concept

Key Idea: K-means forms clusters as Voronoi regions. Each cluster contains points that are closest to its mean (centroid).

Important Property: Cluster boundaries are:

  • Linear (straight lines / hyperplanes)
  • Formed by intersection of half-spaces

Thus, K-means creates: Convex regions; Linearly separable clusters

Mathematical Foundation (Case: K=2)

Assignment Rule: A point x belongs to cluster 1 if:

‖x−μ₁‖² ≤ ‖x−μ₂‖²

Linear Form (After Expansion):

xᵀ(μ₂−μ₁) ≤ ½(‖μ₂‖² − ‖μ₁‖²)

Insight: This is a linear inequality → boundary is a straight line (hyperplane).

Geometric Interpretation

  • Boundary = Perpendicular bisector
  • Divides space into two equal parts
  • Points on boundary are equidistant from both means

Midpoint Property: The midpoint ½(μ₁ + μ₂) always lies on the boundary.

General Case (K Clusters)

Region Formation: Each cluster is defined as points closer to its mean than all others.

Mathematical Idea: Region = intersection of K−1 half-spaces.

Voronoi Region: Each cluster forms a Voronoi cell, defined by boundaries between pairs of means.

Example (K=3)
Means: μ₁, μ₂, μ₃
Boundaries:
  • Between μ₁ & μ₂
  • Between μ₁ & μ₃
  • Between μ₂ & μ₃
Final cluster region = intersection of relevant boundaries.

Limitations of K-Means

Key Limitation: K-means can only create: Linear boundaries; Convex clusters

Example: Concentric Circles
Data: inner circle + outer circle. What K-Means Does: Draws a straight line, splitting circles into halves. Why It Fails: True clusters are non-linear and non-convex. K-means cannot detect such shapes.

Advanced Concept: Kernel K-Means

Idea: Use the kernel trick to handle non-linear data.

How it Works

  • Map data to higher-dimensional space
  • Apply K-means there

Key Insight: Data may become linearly separable in higher dimensions.

Result: Can capture non-linear cluster structures.

Related Algorithm: Spectral Clustering – uses eigenvectors of the kernel matrix.

Quick Summary:
  • K-means forms Voronoi regions
  • Boundaries are linear, perpendicular bisectors
  • Each cluster = intersection of half-spaces
  • Works well for convex, spherical clusters
  • Fails for non-linear shapes (e.g., circles)
  • Solution: Kernel K-means

Lecture 5: Initialization of Centroids, K-means++

Core Concepts

Importance of Initialization

  • K-means always converges, but the final result depends heavily on the starting point.
  • Poor initialization ⇒ bad clusters (local minima).

Local Minima Problem

  • Algorithm may stop at suboptimal clustering.
  • Reason: No point wants to move, but a better global arrangement exists.

Initialize Means vs Partitions

  • Better approach: initialize centroids (means) directly.
  • Then assign each point to nearest mean.
  • More stable than random cluster assignment.

Initialization Strategies

Strategy A: Uniform Random Initialization

Process: Randomly pick K data points as initial centroids.

Improvement Trick: Run K-means multiple times, choose solution with lowest cost.

Pros & Cons:

  • Fast and simple
  • Works well for large datasets
  • Can be unlucky
  • May lead to poor clustering

Strategy B: K-means++

Idea: Choose centroids that are far apart from each other.

Key Feature:

  • Means selected one by one (iteratively)
  • Uses probability-based selection

Advantage:

  • Better initialization ⇒ better clustering
  • Avoids poor starting points

K-means++ Algorithm

Step 1: First Mean: Pick randomly from dataset.

Step 2: Scoring Points: Score of a point = distance to nearest existing mean:

score(x) = minj∈{1,…,l−1} ‖x − μⱼ‖²

Points far from current means get high score.

Step 3: Probability Selection: Select next mean with probability:

P(x) = score(x) / Σ all scores

Far points ⇒ higher probability.

Examples

Example 1: Random Initialization
Dataset: 1000 points, K=5.
Randomly pick: x₅, x₇₂, x₈₉, x₂₀₀, x₂₂.
These act as initial centroids.
Example 2: Probability Calculation
Scores: 10, 20, 30 → Total = 60.
Probabilities:
  • Point 1: 10/60 = 1/6
  • Point 2: 20/60 = 1/3
  • Point 3: 30/60 = 1/2
Insight: Highest score ⇒ highest chance. But randomness allows exploration and avoids bad cases.

Performance & Guarantees

Objective Function: Minimize clustering cost:

Σᵢ ‖xᵢ − μzᵢ‖²

Theoretical Guarantee (K-means++): Produces solution within a constant factor of optimal. Much better than random initialization.

Computational Cost

  • K-means++: Slower than random; Needs multiple passes over data
  • Random: Faster; Less reliable
Quick Summary:
  • Initialization is crucial in K-means.
  • Bad start ⇒ bad clustering.
  • Methods: Random → fast but risky; K-means++ → smarter and reliable
  • K-means++ idea: pick points far from existing centroids, probability proportional to distance squared.

Lecture 6: Choice of K

The Problem of Choosing K

Key Issue

  • In real-world datasets, the number of clusters K is unknown.
  • Simply minimizing clustering error is not sufficient.
  • Lower cost ≠ better clustering always.

Objective Function

Standard Cost Function:

F = Σᵢ ‖xᵢ − μzᵢ‖²

Measures: total squared distance of points from their cluster means.

The K=n Trap

If K = n:

  • Each point = its own cluster
  • Distance = 0
  • So F = 0 (minimum possible)

Why This is Useless:

  • No grouping
  • No pattern discovery
  • No data compression
  • Clustering becomes meaningless

Solution: Objective + Penalty

Key Idea: Balance between: Fit (low error) and Simplicity (fewer clusters)

Total Cost = Objective Value + Penalty(K)

Trade-off: Increasing K: Decreases objective value; Increases penalty

Decision Rule: Add a new cluster only if reduction in error > penalty increase.

Curve Insight

  • Objective ↓ with K
  • Penalty ↑ with K
  • Combined curve: first decreases, then increases
  • Best K = minimum point of curve

Common Methods to Choose K

A. Akaike Information Criterion (AIC)

AIC = 2K − 2log(likelihood)

Penalty grows linearly with K.

B. Bayesian Information Criterion (BIC)

BIC = K log(n) − 2log(likelihood)

Penalty depends on: Number of clusters K; Number of points n

C. Elbow Method

Idea: Plot K vs objective value. Look for "elbow point" (sharp bend).

Interpretation:

  • Before elbow → big improvement
  • After elbow → small improvement
  • Choose K at elbow.

Summary of Lloyd's Algorithm Concepts

  • Convergence: Always converges, may reach local minimum.
  • Cluster Shape: Forms Voronoi regions, linear boundaries.
  • Initialization: Important for good results.
    • Random (fast)
    • K-means++ (better)
  • Practical Strategy:
    • Try different K values
    • Use multiple initializations
    • Compute cost for each
    • Apply Elbow / AIC / BIC
    • Choose best clustering

Quick Summary:
  • K is usually unknown.
  • K=n gives zero cost but useless result.
  • Solution: add penalty for large K.
  • Methods: AIC, BIC, Elbow method.
  • Best K: balance between accuracy & simplicity.
