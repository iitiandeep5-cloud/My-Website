Week 1: Short Notes

Lecture 1: Introduction to Machine Learning

What is Machine Learning?

  • Definition: Study of algorithms that learn from data and make predictions.
  • Key Idea: Instead of explicitly programming rules, ML allows computers to learn patterns from data.
  • Foundation: Driven by mathematics (linear algebra, probability, optimization).

ML vs Artificial Intelligence (AI)

  • Artificial Intelligence (AI): Broad concept of making machines behave like humans.
  • Machine Learning (ML): Subset of AI; machines learn from data.
  • Deep Learning: Subset of ML; uses neural networks.

Hierarchy: AI ⊃ ML ⊃ Deep Learning

What Machine Learning is NOT

Not Procedural Programming:

  • Traditional programs follow fixed rules.
  • ML systems learn rules automatically from data.
  • Example: Tax calculator = fixed rules; Spam filter = learned patterns.

Not Memorization:

  • Memorizing training data ≠ learning.
  • A good model must generalize to unseen data.
  • Example: Memorizing 2000 tree images ≠ understanding "tree".

Key Concept: Generalization

  • Definition: Ability to perform well on unseen data.
  • Goal of ML: Build models that generalize beyond training data.
  • Good ML Model:
    • Works well on training data.
    • Works well on new, unseen data.

Learning by Experience

ML models improve by:

  • Making predictions.
  • Receiving feedback.
  • Adjusting behavior.

Example: A robot hits a wall → learns to avoid it next time.

Mathematical Foundation

  • Linear Algebra
  • Probability
  • Optimization

Real-Life Applications of ML

Human-Like Abilities

  • Computer Vision → Object recognition (faces, trees).
  • Speech & Text Processing → Language, accents, handwriting.

Everyday Digital Use

  • E-commerce → Product recommendations.
  • OTT Platforms → Personalized movie suggestions.
  • Spam Filters → Detect spam emails/SMS.
  • Social Media → "People you may know."

Prediction & Forecasting

  • Weather Prediction → Rainfall forecasts.
  • Stock Market → Price movement prediction.
  • Sports Analytics → Match winner prediction.

Technical & Scientific Use

  • Signal Processing → Separate vocals from music.
  • Photo Organization → Group images by faces.
  • Robotics → Navigation through trial & error.
  • Scientific Research → Analyze experimental data.
Final Summary:
  • ML = Learning from data, not hardcoded rules.
  • Goal = Generalization (work on unseen data).
  • Not memorization, not fixed programming.
  • Improves through experience & feedback.
  • Applications span digital platforms, finance, science, and robotics.

Lecture 2: Paradigms of Machine Learning

Supervised Learning (Learning with Labels)

  • Definition: Data has inputs (features) and outputs (labels). Goal is to learn mapping input → output.
  • Types:
    • Classification (Discrete Output):
      • Binary (2 classes) → e.g., Spam vs Not Spam
      • Multi-class (>2 classes) → e.g., Digit recognition (0–9)
    • Regression (Continuous Output): Predict real values (e.g., rainfall, house price).
    • Ranking / Ordinal Learning: Ordered outputs (e.g., movie recommendations).
    • Structured Learning: Outputs with structure (trees, graphs, sentence parsing).

Unsupervised Learning (No Labels)

  • Definition: Only data given, no correct answers. Goal is to discover hidden patterns.
  • Types:
    • Clustering: Group similar data points (e.g., photos → humans, animals, objects).
    • Representation Learning: Learn better features automatically (e.g., raw pixels → edges, shapes).

Sequential Learning (Learning Over Time)

  • Definition: Learning step-by-step, updating continuously with feedback.
  • Types:
    • Online Learning (Full Feedback): Feedback for chosen and all other actions (e.g., stock prediction with expert advice).
    • Multi-Armed Bandits (Partial Feedback): Feedback only for chosen action (e.g., medical treatment outcome).
    • Reinforcement Learning (RL):
      • Definition: Agent interacts with environment, learns via trial & error with rewards.
      • Key Idea: Policy = mapping from state → action.
      • Examples: Robot navigation, game playing (chess, RL agents).

Key Concepts

  • Best Model: Find model that minimizes error or maximizes accuracy.
  • Linear Relationship: Fit line h = wx + b to represent data (choose best fit).
  • Uncertainty / Noise: Real-world data imperfect → handle with probability.
  • Policy (RL): Decision rule mapping state → action.

Real-Life Applications

  • Supervised Learning: Spam detection, rainfall prediction, friend suggestions.
  • Unsupervised Learning: Voice separation, photo clustering.
  • Sequential Learning: Stock market prediction, medical treatment selection.
  • Reinforcement Learning: Robotics, game playing.

Mathematical Foundations

  • Linear Algebra: Vectors, matrices, data representation.
  • Probability & Statistics: Handle uncertainty and randomness.
  • Calculus & Optimization: Find best parameters (min/max problems).

Recommended Books

  • Linear Algebra and Applications – Gilbert Strang
  • A First Course in Probability – Sheldon Ross
  • Pattern Recognition and Machine Learning – Christopher M. Bishop
  • Mathematics for Machine Learning
Final Summary:
  • Supervised → labeled data (classification, regression).
  • Unsupervised → no labels (clustering, representation).
  • Sequential → learning over time (online, bandits).
  • Reinforcement → trial & error with rewards.
  • Goal = best model under uncertainty.
  • Core math = Linear Algebra + Probability + Optimization.

Lecture 3: Representation Learning: Part 1

Definition

  • Representation Learning: A subfield of Unsupervised Learning.
  • Goal: Learn useful structures or features from data without labels.
  • Idea: Instead of working with raw data, learn a better representation.

Data Representation

  • Each data point can be represented as a vector in d-dimensions.
  • Example: Person data (height, weight, age) → point in R³.

Core Idea: Compression = Learning

  • Principle: "Comprehension is compression."
  • Meaning: If you can compress data, you have understood it.
  • Why: Compression retains important information, removes redundancy, and reveals structure.

Case 1: Exact Reconstruction (Perfect Structure)

  • Situation: Data lies exactly on a line.
  • Example: Points follow relation y = 2x.
  • Naive Storage: For n points in d dimensions → store d×n values.
  • Compressed Representation:
    • Store one representative vector w.
    • Store one scalar per point cᵢ.

Reconstruction Formula: xᵢ = cᵢ · w

Example: If w=[1,2], c=2.5 → x=[2.5,5].

Storage Reduction: Before: d×n. After: d+n.

Note: Representative vector w is not unique; any vector along the same line works.

Case 2: Approximate Reconstruction (Real Data)

  • Problem: Real-world data is not perfectly on a line.
  • Solution: Use approximation via projection.

Projection Idea: Replace point x with its closest point on the line.

Optimization Goal: c* = argminc ‖x − cw‖²

Optimal Solution: c* = xᵀw / ‖w‖²

Where: xᵀw = dot product, ‖w‖² = squared length of vector.

Special Case (Unit Vector): If ‖w‖ = 1, then c* = xᵀw

Final Projection Formula: x̂ = (xᵀw)w

Intuition

  • Original point = x.
  • Line direction = w.
  • Projection = shadow of x on line.
  • Error = distance between x and projected point.

Real-Life Interpretation

  • Image compression → reduce pixels but preserve features.
  • PCA (Principal Component Analysis) → find best directions.
  • Noise removal → keep signal, remove randomness.
Final Summary:
  • Representation Learning = learn useful structure without labels.
  • Key idea = Compression = Understanding.
  • If data lies on a line: xᵢ = cᵢw
  • If not: Use projection.
  • Best coefficient: c* = xᵀw / ‖w‖²
  • With unit vector: c* = xᵀw
  • Projection formula: x̂ = (xᵀw)w

Lecture 4: Representation Learning: Part 2

Core Problem

  • Reality Check: Real-world data is messy; points do not lie perfectly on a line.
  • Example: Height vs Weight → shows a trend, not an exact line.
  • Goal: Find a proxy line that compresses data (2D → 1D) while preserving maximum information.
  • Best Line Criteria: Among all possible lines, choose the one with minimum reconstruction error.

Mathematical Setup

  • Dataset: x₁, x₂, …, xN ∈ Rd
  • Line Representation: Line defined by vector w.
  • Constraint: unit vector wᵀw = 1

Projection (Proxy Point): x̂ᵢ = (xᵢᵀw)w

Objective Function (Minimize Error)

  • Error: Distance between actual point and projected point.

Average Reconstruction Error: f(w) = (1/N) Σᵢ ‖xᵢ − (xᵢᵀw)w‖²

Key Simplification

After simplification: (1/N) Σᵢ (‖xᵢ‖² − (xᵢᵀw)²)

  • Insight: ‖xᵢ‖² is constant (independent of w).

Equivalent Problem: max‖w‖=1 (1/N) Σᵢ (xᵢᵀw)²

  • Interpretation: Maximize variance along direction w.

Covariance Matrix

Definition: C = (1/N) Σᵢ xᵢxᵢᵀ

Final Objective: max‖w‖=1 wᵀCw

Final Solution

  • Best Direction w: Eigenvector of C corresponding to the largest eigenvalue.
  • Reasoning:
    • Eigenvector → direction of maximum variance.
    • Eigenvalue → amount of variance.
    • Largest eigenvalue → most important direction.

Real-Life Example (Height & Weight)

  • Scenario: 100 people with features: height and weight.
  • Problem: Points do not lie on an exact line.
  • PCA Solution: Finds best line such that:
    • Distance from all points to line is minimized.
    • Maximum trend captured.
  • Result: Convert (height, weight) → single number representing position along best line.
Final Summary:
  • Real data ≠ perfect line.
  • Goal: Find best proxy line.
  • Objective: Minimize reconstruction error ⇔ Maximize variance.
  • Key Formulas: Projection: x̂ = (xᵀw)w
  • Covariance Matrix: C = (1/N) Σ xᵢxᵢᵀ
  • Final Problem: max wᵀCw, subject to ‖w‖=1
  • Final Answer: Best direction = Eigenvector of C with largest eigenvalue.

Lecture 5: Representation Learning: Part 3

Core Problem: Why One Line is Not Enough

  • Limitation: A single line captures only one dimension.
  • Reality: Real data may lie in higher dimensions.
  • Example: Data in 3D forming a plane (2D structure). One line captures only part of it; remaining information is lost as error.
  • Goal: Capture more structure using multiple directions (lines).

Key Idea: Residue (Error) = Information

Residue Definition: r = x − (xᵀw)w

  • Insight: Residue is not just noise. If residues align, hidden structure exists.
  • Example:
    • First line captures main trend.
    • Residues align along second direction → another line.

Important Formulae

  • Projection (Proxy): x̂ = (xᵀw)w
  • Mean (Center of Data): μ = (1/n) Σᵢ xᵢ
  • Centered Data: xcentered = xᵢ − μ
  • Optimization Goal: max‖w‖=1 wᵀCw, where C = covariance matrix. Solution = eigenvector of C.

Iterative Algorithm (Multi-Dimensional Learning)

  • Step 1: Find first direction w₁ → captures maximum variance.
  • Step 2: Compute residues → remaining information.
  • Step 3: Create new dataset = residues.
  • Step 4: Find second direction w₂ → captures next most variance.
  • Step 5: Repeat for more dimensions.

Result: Instead of one line, obtain multiple lines forming a basis of data structure. This is essentially Principal Component Analysis (PCA).

Importance of Data Centering

  • Problem: If data is not centered, algorithm may focus on location instead of direction.
  • Solution: Shift data so center = origin.

Formula: xcentered = x − μ

Intuition: Centering ensures we capture spread (variance), not just position.

Example: 3D Plane Data

  • Scenario: Points lie on a plane in 3D.
  • Single Line: Captures only one direction, loses rest of plane.
  • Multi-Line Approach:
    • Line 1 → main direction.
    • Residue → second direction.
    • Together → represent plane.

Final Representation: Each point = combination of coefficient along w₁ and coefficient along w₂. Thus, data is reduced from 3D → 2D (compressed).

Big Picture Intuition

  • Data = combination of important directions.
  • Each new direction explains remaining variance.
  • Residue = what is still unexplained.

Open Questions

  • How to compute eigenvectors?
  • How many components to keep?
  • Where exactly is compression happening?
  • What features are being learned?

These lead to the full PCA theory.

Final Summary:
  • One line is not enough for real data.
  • Use residue to find more structure.
  • Iteratively find w₁, w₂, w₃, …
  • Key Ideas: Projection (xᵀw)w, Residue x−(xᵀw)w, Centering x−μ, Objective: maximize variance.
  • Final Insight: PCA = find multiple orthogonal directions capturing maximum variance step-by-step.

Lecture 6: Principal Component Analysis: Part 1

Core Objective

  • Goal: Find directions (lines) that minimize reconstruction error ⇔ maximize variance.
  • Intuition: The best line captures maximum information.

Iterative PCA Algorithm

  • Step 1: Find First Direction (w₁)
    • Given centered data x₁, x₂, …, xn ∈ Rd:
    • w₁ = argmax‖w‖=1 wᵀCw, where C = (1/n) Σᵢ xᵢxᵢᵀ
  • Step 2: Compute Residues xᵢ′ = xᵢ − (xᵢᵀw₁)w₁ — Removes information captured by w₁.
  • Step 3: Find Next Direction (w₂) using residue dataset: w₂ = argmax‖w‖=1 wᵀC′w
  • Repeat for k steps: Obtain directions w₁, w₂, …, wk.

Key Concepts

  • Orthogonality: Residues are perpendicular to previous direction. w₁⊥w₂⊥w₃…
  • Orthonormality: wᵢᵀwⱼ = 0 (i ≠ j), ‖wk‖ = 1. Meaning: All directions are perpendicular and unit length, forming an orthonormal basis.

Change of Basis

  • Instead of standard axes, use new axes w₁, w₂, …, wd.

Reconstruction Formula: xᵢ = Σⱼ (xᵢᵀwⱼ)wⱼ

Interpretation: Each point is a combination of principal directions.

Data Compression

  • When Compression Happens: Data lies in a low-dimensional subspace.
  • Storage Comparison:
    • Original: d×n.
    • PCA: (k×d) + (k×n).

Example: n=100, d=100, k=3. Before: 100×100 = 10,000 values. After: (3×100)+(3×100) = 600 values. Result: Massive compression.

When to Stop?

  • Problem: Residue rarely becomes exactly zero.

Pythagoras Relation: ‖xᵢ‖² = ‖xᵢ − (xᵢᵀw)w‖² + (xᵢᵀw)²

Meaning: Total variance = error (residue) + captured variance.

  • Stopping Rules:
    1. Elbow Method: Plot variance vs number of components; stop where curve flattens.
    2. Information Threshold: Choose k such that ~95% variance is captured.

Intuition Summary

  • Each step captures the most important direction.
  • Removes it, then focuses on remaining information.
Final Summary:
  • PCA finds orthogonal directions of maximum variance.
  • Uses projection, residue, and iteration.
  • Key Results: Best direction = eigenvector of covariance matrix. Directions are orthonormal. Data = combination of these directions.
  • Final Insight: PCA rotates the coordinate system to align with maximum variance directions.

Lecture 7: Principal Component Analysis: Part 2

Core Concepts

Covariance Matrix (C): C = (1/n) Σᵢ xᵢxᵢᵀ — captures relationships between features, defined for centered data.

  • Principal Components: Directions w₁, w₂, …, wk.
    • Properties: Orthonormal (perpendicular + unit length).
    • Capture maximum variance step-by-step.
  • De-correlation: PCA transforms data into new features that are independent/uncorrelated. Each new direction adds new information.
  • Dimensionality Reduction: Converts high-dimensional data to low-dimensional form. Rd → Rk, k ≪ d

Linear Algebra & Formulae

Eigenvalue Equation: Cwk = λkwk

  • wk = eigenvector (direction).
  • λk = eigenvalue (importance).

First Principal Component: w₁ = eigenvector with largest eigenvalue → Most important direction.

Variance Interpretation: λk = wkᵀCwk = (1/n) Σᵢ (xᵢᵀwk)²

  • Eigenvalue = variance captured.
  • Always ≥ 0 → covariance matrix is positive semi-definite.

Key Principles

  • Error vs Variance: Minimizing reconstruction error ⇔ maximizing variance.
  • Distinguishing Capability:
    • Good projection: points are spread out.
    • Bad projection: points collapse together → loss of information.

Choosing Number of Components (k)

Rule of Thumb: (Σᵢ₌₁ᵏ λᵢ) / (Σᵢ₌₁ᵈ λᵢ) ≥ 0.95

Meaning: Keep enough components to retain ≥ 95% variance.

Examples

  • Geometric Insight:
    • Good line → spreads data.
    • Bad line → compresses data, loses information.
  • Height & Weight Example:
    • Original features: height, weight (correlated).
    • PCA transformation: (height+weight)/2, (weight−height)/2
  • Result: new features are uncorrelated → easier for ML models to learn.

Eigenfaces (Real Application)

PCA used in face recognition → learns important face patterns, called eigenfaces.

Final Summary:
  • PCA = Dimensionality Reduction technique.
  • Uses covariance matrix, eigenvectors, and eigenvalues.
  • Key Ideas: Eigenvector → direction. Eigenvalue → variance (importance). Keep top k components.
  • Final Insight: PCA finds new axes where data is most spread out and independent.
