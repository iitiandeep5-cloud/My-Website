Week 8: Short Notes

Lecture 1: Generative Model-Based Algorithm

Generative Models (Core Idea)

What is a Generative Model?

A generative model learns how data is generated: P(X,Y). It models features X and labels Y together (jointly).

Generative vs Discriminative

TypeWhat it learns
DiscriminativeP(Y | X)
GenerativeP(X,Y)

Joint Distribution Decomposition

P(X,Y) = P(Y)·P(X|Y)

This is the key idea used in generative models.

Goal:

  • Learn P(Y) → probability of label (Spam/Not Spam)
  • Learn P(X|Y) → probability of features given label

Feature Representation (Binary Encoding)

  • Problem: Emails are text → need numeric format.
  • Solution: Binary Vector
    • Dictionary size = D
    • Email → vector of size D

Encoding Rule:

xᵢ = 1 if word present; 0 if word absent

Example
Email: "hello how are you"
Dictionary: [about, are, hello, how, you, zebra]
WordValue
about0
are1
hello1
how1
you1
zebra0

Generative Story (Model Intuition)

Step 1: Choose Label

  • Toss a biased coin: P(Y=1) = p(Spam), P(Y=0) = 1−p(Not Spam)

Step 2: Generate Email

  • Sample from P(X|Y)
  • If Spam → use spam distribution
  • If Not Spam → use normal distribution

Dice Analogy

Imagine two rooms:

  • Spam Room
  • Non-Spam Room

Inside each room: a 2D-faced die. Each face = one possible email pattern.

Example (D = 3)
Possible emails: 000, 001, 010, 011, 100, 101, 110, 111
Total = 2³ = 8 patterns

Spam room → higher probability for words like "money", "lottery"
Non-spam room → lower probability for same

Model Parameters

To learn the model, we need probabilities:

  1. Label Probability P(Y) → only 1 parameter
  2. Feature Probabilities for each label → 2D−1 parameters

Total Parameters: 1+(2D−1) + (2D−1) = 2D+1−1

Big Problem: Parameter Explosion

Example: If D=10,000: Total parameters ≈ 210000

  • Extremely huge number
  • Impossible to store
  • Impossible to compute
  • Cannot train even with powerful GPUs

Conclusion

  • The model is theoretically correct
  • Practically impossible due to parameter explosion

Final Insight: We need simpler assumptions and fewer parameters → leads to models like Naive Bayes.

Lecture 2: Alternate Generative Model-Based Algorithm

Optimization of Generative Models: Naive Bayes

Core Problem: Parameter Explosion

Issue in Basic Generative Model:

  • For D binary features (words):
    • Total possible emails: 2D
    • Need to learn 2D probabilities

Problem: If D=10,000: 210000 (impossible)

Cannot: Store, Compute, Train

Goal: Reduce parameters while still modeling data.

Key Idea: Class Conditional Independence

Assumption: Given class Y, all features are independent.

P(X|Y) = P(f₁, f₂,...,f_D | Y)

Becomes:

P(X|Y) = Π P(fᵢ | Y), for i = 1 to D

Analogy:

  • Instead of one big dice → use D independent coins
  • Each word has its own probability
  • Words don't affect each other

Model Parameters

Total Parameters: 2D + 1

Breakdown:

  1. Label Probability P(Y=1) = p, P(Y=0) = 1−p → 1 parameter
  2. Feature Probabilities
    • For Spam (Y=1): P₁ᵢ = P(fᵢ=1 | Y=1)
    • For Non-Spam (Y=0): P₀ᵢ = P(fᵢ=1 | Y=0)

Total = D + D = 2D

Key Formulae

Step 1: Class Probability P(Y=1) = p, P(Y=0) = 1−p

Step 2: Class Conditional Probability

P(X|Y=y) = Π Pyᵢ^fᵢ (1−Pyᵢ)^(1−fᵢ), for i = 1 to D

Where: fᵢ = 1 → word present, fᵢ = 0 → word absent

Step 3: Joint Probability P(X,Y) = P(X|Y)·P(Y)

Examples

Example A: Word Behavior
  • Word: "Lottery"
    • Spam: P11=0.8
    • Non-Spam: P01=0.1
    • → Highly indicative of spam
Example B: Email Probability
D=3, Email = (0,1,1), Class = Spam (Y=1)

Step-by-Step:

  • Word 1 absent: (1−P11)
  • Word 2 present: P12
  • Word 3 present: P13
Final Probability: P(X|Y=1) = (1−P11)·P12·P13

Parameter Comparison

ModelParameters
Original2D ✗
Naive Bayes2D+1 ✓

Huge reduction in complexity.

Parameter Estimation

Method Used: Maximum Likelihood Estimation (MLE)

Idea: Estimate probabilities from data based on frequency.

P1ᵢ = (# spam emails with word i) / (total spam emails)

Final Intuition

  • Instead of modeling full email patterns → model each word independently
  • Combine probabilities to approximate joint distribution

Lecture 3: Naive Bayes Algorithm

Naive Bayes Algorithm (Binary Classification)

Key Concepts & Assumptions

Generative Model

  • Assumes data is generated as follows:
    • Choose label Y (Spam / Not Spam)
    • Generate features X based on Y

Class Conditional Independence

P(f₁,f₂,...,f_D | Y) = Π P(fⱼ | Y), for j = 1 to D

  • Each feature is independent given the class.
  • Reduces complexity from exponential (2D) to linear (2D+1).

Reality Check:

  • Words like "Lottery" and "Money" may be related.
  • Despite this, the model works well in practice.

Parameters to Estimate

Total Parameters: 2D+1

Types:

  1. Prior Probability P(Y=1) = p — Probability of spam.
  2. Feature Probabilities Py,j = P(fⱼ = 1 | Y = y)
    • For Spam (y=1): P1,1,...,P1,D
    • For Non-Spam (y=0): P0,1,...,P0,D

Parameter Estimation (MLE)

Method: Maximum Likelihood Estimation (MLE)

(A) Estimate Prior:

P̂ = (1/n) Σ yᵢ, for i = 1 to n

Fraction of spam emails.

(B) Estimate Feature Probability:

P̂y,j = (# samples where fᵢ,ⱼ=1 and yᵢ=y) / (# samples where yᵢ=y)

  • Numerator → times word appears in class y
  • Denominator → total samples of class y

Prediction Formula

Goal: Predict label for new data xtest.

Decision Rule:

ŷ = 1 if P(Y=1|Xtest) > P(Y=0|Xtest); else 0

Using Bayes Rule: Compare numerators only.

For y=1 (Spam): [ Π P̂1,ⱼ^fⱼ (1−P̂1,ⱼ)^(1−fⱼ) ] × P̂

For y=0 (Not Spam): [ Π P̂0,ⱼ^fⱼ (1−P̂0,ⱼ)^(1−fⱼ) ] × (1−P̂)

Choose the larger value.

Illustrative Example (Spam Detection)

Data Representation
Email → binary vector. Example: "Lottery win now" → [1,0,1,0,...]

Estimation Example:

  • Total emails = 1000, Spam = 600

P̂ = 0.6

Word "Lottery" in spam:

P̂1,j = (# spam emails with "Lottery") / 600

Prediction Example:

  • For a new email:
    • Compute probability for Spam
    • Compute probability for Not Spam
    • Compare both
  • If P(Spam) > P(Not Spam) → Predict Spam

EstimationMLE (counting)
PredictionBayes Rule
Use CaseSpam filtering
One-Line Intuition: Naive Bayes predicts by multiplying independent feature probabilities and choosing the most likely class.

Lecture 4: Pitfalls of Naive Bayes Algorithm

Zero Frequency Problem

  • Definition: Occurs when a word in test data was never seen in training.
  • Reason: Using MLE: P̂1ⱼ = 0, P̂0ⱼ = 0, because the word never appeared in training data.

Impact: Naive Bayes uses multiplication of probabilities:

P(Y|X) ∝ Π (P̂yᵢ)^fᵢ (1−P̂yᵢ)^(1−fᵢ), for i = 1 to D

If any one term = 0 → whole product = 0.

Result:

  • Both Spam & Non-Spam probabilities = 0
  • Cannot compare → model fails

Mathematical View

Spam Probability:

P(Y=1|X) ∝ [ Π (P̂1ᵢ)^fᵢ (1−P̂1ᵢ)^(1−fᵢ) ] × P̂

Problem Case: If fᵢ = 1 (word present) and P̂1ᵢ = 0: (P̂1ᵢ)^fᵢ = 0¹ = 0

Final result: P(Y=1|X) = 0

Example: "Zebra" Case
  • Situation: Word "zebra" not in training data → P̂1,zebra = 0.
  • Test Email: Contains "Lottery", "Money", "Zebra".
  • Outcome: Even if strong spam words exist, P(Spam)=0
Model ignores useful information → wrong decision.

Solution: Laplace Smoothing

  • Idea: Never allow probability = 0.
  • Trick: Add +1 to every count.

Formula:

P̂y,j = (count of word j in class y + 1) / (total words in class y + 2)

Intuition: Pretend every word appeared at least once.

Effect:

  • No zero probabilities
  • Small but non-zero values

Before vs After Smoothing

CaseWithout SmoothingWith Smoothing
New word0 ✗Small value ✓
PredictionBreaks ✗Works ✓

Numerical Stability (Log Trick)

Problem: Multiplying many small probabilities → underflow.

0.001 × 0.002 × 0.0001 ≈ 0

Solution: Take logarithm. log(ab) = log a + log b

New Formula:

log P(Y|X) = Σ log[(P̂yᵢ)^fᵢ (1−P̂yᵢ)^(1−fᵢ)] + log P(Y), for i = 1 to D

Benefit:

  • Avoids very small numbers
  • Faster computation

Important: Log does not fix zero problem (log(0) undefined). → Always apply Laplace smoothing first, then use log.

One-Line Intuition: One unseen word can destroy Naive Bayes prediction — Laplace smoothing prevents this by ensuring no probability is ever zero.

Lecture 5: Decision Function of Naive Bayes

Key Concepts

  • Decision Function: The rule used to classify a new data point xtest as class 0 or 1.
  • Generative vs Discriminative:
    • KNN & Decision Trees → discriminative
    • Naive Bayes → generative
  • Linearity: The decision function of Naive Bayes is linear; the boundary is a hyperplane in feature space.
  • Binary Features: Assumes features f₁,f₂,...,f_d are 0/1 (common in text classification).
  • Log-Likelihood Ratio: Converting product of probabilities into sums makes math easier and reveals the linear structure.
  • Bias (b): Shifts the decision boundary from the origin; depends on priors and feature-independent terms.

Mathematical Formulae

Decision Rule:

Predict class 1 if: P(ytest=1 | Xtest) ≥ P(ytest=0 | Xtest)

Equivalently, using log-ratio:

log [ P(ytest=0|Xtest) / P(ytest=1|Xtest) ] ≥ 0

Using Bayes' Rule:

log [ P(Xtest|y=0)P(y=0) / P(Xtest|y=1)P(y=1) ] ≥ 0

Linear Form: wᵀXtest + b ≥ 0

Where:

wᵢ = log [ P̂0,ᵢ(1−P̂1,ᵢ) / P̂1,ᵢ(1−P̂0,ᵢ) ]

b = Σ log[(1−P̂0,ᵢ)/(1−P̂1,ᵢ)] + log[(1−P̂)/P̂], for i = 1 to d

  • wᵢ → weight for feature i
  • b → bias (from priors and feature-independent terms)
  • P̂1,ᵢ → probability of feature i=1 given class 1
  • P̂ → prior probability of class 1

Examples

Spam Classification
  • Features: Binary (word present/absent)
  • Decision function: wᵀXtest + b ≥ 0 ⇒ Spam. Otherwise → Not Spam
Tie-Breaking: If decision = 0 → default to Spam (common in spam filters).

Comparison of Boundaries

ModelBoundary Type
KNN (small K)Irregular, many holes
KNN (large K)Smoothed but nonlinear
Decision TreeAxis-aligned steps
Naive BayesSingle linear hyperplane

Related Concepts

  • Laplace Smoothing: Handles zero probabilities.
  • Non-Binary Features: Can generalize to continuous features using Gaussian Naive Bayes.
Key Intuition: Naive Bayes produces a straight-line decision boundary in feature space (linear) due to the log transformation of probabilities, even though it is a generative model.

Lecture 6: Gaussian Naive Bayes

Core Concepts

  • Transition to Real Features: Unlike binary Naive Bayes, GNB works with real-valued features X ∈ ℝd.
  • Generative Modeling:
    • Model label probabilities P(Y)
    • Model feature distributions given labels P(X|Y)
  • Gaussian Assumption: For each class y, features follow a Normal (Gaussian) distribution.
  • Covariance Assumption: Often assume shared covariance matrix Σ for both classes; means μ₀ and μ₁ may differ.
  • Decision Boundary:
    • Linear → if covariances are the same.
    • Quadratic → if covariances differ.

Key Parameters & Formulae

Parameters to Estimate (via MLE):

ParameterDescription
p̂Probability of label 1
μ̂₁Mean vector of class 1
μ̂₀Mean vector of class 0
Σ̂Shared covariance matrix

Maximum Likelihood Estimates (MLE):

Class Probability:

p̂ = (# of points with y=1) / (total points)

Class Means:

μ̂y = (1/ny) Σ xᵢ, for i where yᵢ = y

Shared Covariance Matrix:

Σ̂ = (1/n) Σ (xᵢ−μ̂yᵢ)(xᵢ−μ̂yᵢ)ᵀ, for i = 1 to n

Prediction / Decision Function (Linear Decision Rule, shared covariance):

Predict class 1 if: wᵀXtest + b ≥ 0

Weight Vector:

wᵀ = (μ̂₁−μ̂₀)ᵀ Σ̂⁻¹

Bias Term:

b = −½(μ̂₁ᵀΣ̂⁻¹μ̂₁ − μ̂₀ᵀΣ̂⁻¹μ̂₀) + log[p̂/(1−p̂)]

Examples & Geometric Interpretation

  • Density Contours: Gaussian distributions form ellipses (or circles if covariance is identity) around class means.
  • Identity Covariance:
    • Contours are circular.
    • Decision boundary → perpendicular bisector of μ₀ and μ₁, shifted by prior p̂.
  • Effect of Prior p̂:
    • Higher p̂ → decision boundary shifts toward class 0, enlarging region predicting class 1.
  • Different Covariances:
    • If classes have different shapes (elliptical vs circular), linear boundary fails → need quadratic boundary.
Key Intuition: Gaussian Naive Bayes extends the simple Naive Bayes to real-valued features, using Gaussian distributions.
  • Shared covariance → linear boundary.
  • Different covariance → quadratic boundary.
