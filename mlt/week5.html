Week 5: Short Notes

Lecture 1: Supervised Learning

Core Concept

  • Definition: Supervised Learning is a type of Machine Learning where the model learns from labeled data (input + correct output).
  • Analogy: Like a teacher giving questions (inputs) along with answers (labels), and the student learns patterns.

Dataset Structure

Data Points (Inputs): x₁, x₂, …, xn where each x ∈ Rd (Each point lies in a d-dimensional space)

Features: Characteristics of data points (attributes, predictors, variables)

  • Examples: Email → words, length, sender; Image → pixels

Labels (Outputs): y₁, y₂, …, yn — Correct answers for each input

Goal of Supervised Learning

  • Learn a function: f(x) → y
  • Purpose: Predict labels for new unseen data

Classification

🔹 Binary Classification

  • Definition: Output has only two possible classes
  • Label Representation: {0,1} or {+1,−1}
  • Example: Spam Detection
    • Input: Email features
    • Output: 0 → Not Spam, 1 → Spam

🔹 Multi-class Classification

  • Definition: Output has more than two classes
  • Label Representation: {0,1,2,...,k}
  • Example: Digit Recognition (0–9)
    • Input: Image of a digit
    • Output: One of {0,1,2,...,9}
Quick Summary:
  • Supervised Learning: Learning from labeled data
  • Dataset: Features + Labels
  • Goal: Predict output for new data
  • Function Learned: f(x) = y
  • Types of Classification: Binary → 2 classes (Spam / Not Spam); Multi-class → More than 2 classes (Digits 0–9)

Lecture 2: Linear Regression

Fundamental Concepts

  • Definition: Regression is a type of supervised learning where the output is a continuous value (real number).
  • Analogy: Predicting "how much" or "how many" instead of categories.

Training Data

Dataset: {(x₁,y₁), (x₂,y₂), …, (xn,yn)}

Input (Features): xi ∈ Rd → d-dimensional feature vector

Output (Label): yi ∈ R → Real value (e.g., price, temperature)

Goal

  • Learn a function: h: Rd → R
  • Purpose: Predict output for new unseen inputs.
Example: House Price Prediction
  • Input: Size, location, rooms
  • Output: Price (continuous value)

Measuring Error (Error Functions)

Squared Error (Single Data Point):

(h(xᵢ)−yᵢ)²

Total Squared Error (Training Error):

Σᵢ (h(xᵢ)−yᵢ)²

Why Squared Error?

  • Always non-negative
  • Penalizes large mistakes more
  • Minimum value = 0 (perfect prediction)

Overfitting vs Generalization

  • Memorization: Model simply remembers training data.
  • Overfitting:
    • ✅ Performs very well on training data
    • ❌ Performs poorly on new (test) data
    • Happens when model learns noise instead of pattern
  • Example: Rainfall Prediction
    • Model memorizes last 3 years perfectly
    • Fails to predict tomorrow
  • Solution (Generalization):
    • Restrict model complexity
    • Use simpler, structured functions

Linear Regression

Concept

  • Assumes relationship between input and output is linear.

Linear Hypothesis Function

  • Without Bias: hw(x) = wᵀx
  • With Bias (Intercept): hw(x) = wᵀx + b
  • Analogy: y = mx + c

Objective (Goal of Linear Regression)

Minimize total squared error:

minw∈Rd Σᵢ (wᵀxᵢ − yᵢ)²

Important Insight

  • If error = 0 → Perfect linear relationship
  • Not necessarily overfitting
  • Overfitting occurs only when model is too complex for actual data

Lecture 3: Optimizing the Error Function

Matrix Representation

Components

  • Data Matrix (X): X ∈ RD×n, Each column = one data point
  • Transpose (Xᵀ): Xᵀ ∈ Rn×D, Each row = one data point
  • Label Vector (y): y ∈ Rn×1, Contains all output values
  • Weight Vector (w): w ∈ RD×1, Parameters we want to learn

Idea

  • Prediction for all data points at once: Xᵀw

Objective Function (Error Minimization)

  • Error Vector: Xᵀw − y
  • Loss Function (Squared Error): minw∈RD ‖Xᵀw−y‖²
  • Equivalent Vector Form: (Xᵀw−y)ᵀ(Xᵀw−y)

Key Properties

  • ✅ Unconstrained → w can take any value
  • ✅ Quadratic → contains squared terms
  • ✅ Unique minimum (if conditions satisfied)
Example: Predicting Student Marks
  • Features: Study hours, attendance
  • Output: Marks
  • Matrix X stores all student's data

Solving Using Gradient

  • Gradient (Derivative w.r.t w): ∇w = 2XXᵀw − 2Xy
  • Optimal Condition (Normal Equation): XXᵀw = Xy

Closed-Form Solution

Case 1: Invertible Matrix

  • If XXᵀ is invertible: w* = (XXᵀ)⁻¹Xy

Case 2: Non-Invertible (Pseudo-Inverse)

w* = (XXᵀ)†Xy   where † = Pseudo-inverse

Intuition: When multiple solutions exist, pseudo-inverse picks the simplest (minimum length) solution.

Key Observations

Covariance Connection:

  • XXᵀ acts like a covariance matrix (captures feature relationships).

Linear Regression vs PCA

FeatureLinear RegressionPCA
TypeSupervisedUnsupervised
Uses labels (y)?✅ Yes❌ No
GoalPredict outputReduce dimensions

Important Insights

  • Geometric View: Solution = best-fit projection (line/plane closest to all points).
  • Computational Cost: Matrix inverse is expensive for large data → use Gradient Descent (iterative).
  • Non-linear Relationships: Use feature transformations (x², x³, etc.) but still apply linear regression.
  • Probabilistic View: Assumes noise in data, often modeled as Gaussian distribution.

Lecture 4: Geometric Interpretation of Linear Regression

Core Concept

Key Idea: Xᵀw* represents the projection of the label vector (y) onto the subspace formed by features.

Intuition: We are trying to find the closest possible point to y using only combinations of features.

Example Setup (Easy Visualization)

  • Number of features: d=2 (e.g., height, weight)
  • Number of data points: n=3

Feature Vectors

  • F₁: Heights → vector in R³
  • F₂: Weights → vector in R³

Label Vector

  • y ∈ R³ (e.g., income or health score)

Subspace (Plane Concept)

Spanned Subspace: All linear combinations: aF₁+bF₂. This forms a plane (2D surface) inside 3D space.

Meaning: Every point on this plane = a possible prediction. We can only choose solutions on this plane.

Main Mechanism: Projection

  • Problem: Label vector y is usually not on the plane.
  • Solution: Find the closest point on the plane → projection.

Projection Idea: Drop a perpendicular from y to the plane. The point where it hits = best prediction.

Orthogonality Condition: Error vector: y−Xᵀw* is perpendicular (90°) to the plane.

Important Formulae

  • Optimal Solution: w* = (XXᵀ)†Xy
  • Predicted Point (Projection): Xᵀw*
  • General Point on Plane: Xᵀα
  • Orthogonality Condition: (y−Xᵀα)ᵀ(Xᵀα) = 0

Key Cases

  • Case 1: Normal Case (Most Common): y is outside plane. Prediction = projection onto plane, Error ≠ 0
  • Case 2: Perfect Fit (Zero Error): y lies exactly on plane. Xᵀw* = y, No error at all
Real-Life Example: Predicting Salary
  • Features: Experience, Education
  • Labels: Salary
Model finds: Best combination of features, Closest approximation to actual salaries

Final Intuition (Very Important)

Linear Regression = Treat features as axes forming a space (plane/subspace); Express labels as combination of features; If exact match not possible → choose closest point (projection)

One-Line Summary: Linear regression geometrically finds the projection of labels onto the feature space, giving the closest possible prediction.

Lecture 5: Gradient Descent

Computational Challenges

  • Closed-form solution: w* = (XXᵀ)⁻¹Xy

Problem 1: Large Number of Features (d)

  • Need to invert a d×d matrix
  • Complexity: O(d³)
  • Example: If d=10⁹ → impossible to compute

Problem 2: Large Number of Data Points (n)

  • Need to compute XXᵀ
  • Requires multiplying D×n and n×D matrices
  • If n is huge → computation becomes very slow

Conclusion

Closed-form solution is: Not scalable, Not practical for big data

Gradient Descent (GD)

Iterative optimization method

Objective Function:

F(w) = Σᵢ (wᵀxᵢ − yᵢ)²

Goal: Minimize total squared error

Gradient Formula:

∇F(w) = 2XXᵀw − 2Xy

Update Rule:

wt+1 = wt − ηt∇F(wt)

  • wt: Current weights
  • ηt: Learning rate (step size)

Intuition

  • Move in opposite direction of gradient
  • Gradually reach minimum error

Convexity Advantage

  • Squared error = convex function
  • Only one minimum
  • GD always finds global optimum

Stochastic Gradient Descent (SGD)

Used when n (data points) is very large

Idea: Instead of using all data → use a small random sample (batch)

Example: Total data: 1 billion points, Batch size: k=100

  • Much faster computation per step

Noisy Gradient

  • Gradient based on sample ≠ exact
  • Slightly inaccurate but works well over time

SGD Gradient (Sample-Based):

∇Fsample(w) = 2X̄X̄ᵀw − 2X̄ỹ

Output (Averaging Trick):

woutput = (1/T) Σt=1T wt

Average of all iterations → more stable result

Batch Size Concept

  • Small k: Faster per step, More noisy
  • Large k: More accurate, Slower

Guarantee

  • SGD converges to optimal solution with high probability
  • Faster for large datasets

Lecture 6: Kernel Regression

Motivation (Why Kernel Regression?)

Problem with Linear Regression: Assumes h(x) = wᵀx. Only captures linear relationships

  • Limitations:
    • Cannot handle quadratic terms (x²)
    • Cannot handle interaction terms (x₁·x₂)

Example: Non-linear data → linear model gives poor fit, high error

Solution: Map data to higher-dimensional space where it becomes linear

The Kernel Trick

Idea: Instead of explicitly mapping x → φ(x), use kernel function: K(xᵢ,xⱼ) = φ(xᵢ)ᵀφ(xⱼ)

  • Kernel Matrix: K = XᵀX   Size: n×n & Stores similarity between all pairs of points
  • Key Advantage:
    • Avoids computing high-dimensional features directly
    • Saves time + memory

Key Insight: Span of Data

  • Result: w* = Xα* — Optimal weights lie in span of training data

Intuition: Any component of w outside data space has no effect on predictions. Only need combinations of training points.

Solving for α*

α* = K⁻¹y

Uses kernel matrix instead of feature matrix

Note: Depends on labels (y) → supervised (unlike PCA which is unsupervised)

Prediction Formula

For a new point xtest:

Prediction = Σᵢ αᵢ* K(xᵢ, xtest)

Intuition: K(xᵢ,xtest): similarity; αᵢ*: importance of point. Final prediction = weighted similarity sum

Types of Kernels

  • Linear Kernel: K(xᵢ,xⱼ) = xᵢᵀxⱼ
  • Polynomial Kernel: K(xᵢ,xⱼ) = (xᵢᵀxⱼ+c)d
  • Gaussian (RBF) Kernel: K(xᵢ,xⱼ) = exp(−γ‖xᵢ−xⱼ‖²)

Key Advantages

  • Captures non-linear relationships
  • No explicit feature transformation needed
  • Works efficiently in high-dimensional spaces

Final Intuition

Kernel Regression =

  • Convert non-linear problem → linear (in hidden space)
  • Use similarity instead of explicit features
  • Predict using weighted similarity of training points

Alpha Solutionα* = K⁻¹y

One-Line Summary: Kernel regression solves non-linear problems by using similarity (kernels) instead of explicit features.

Lecture 7: Probabilistic View of Linear Regression

Core Concept

  • Linear Regression can be seen as a probabilistic estimation problem.
  • Instead of just fitting a line, we assume data is generated by a process with randomness (noise).

Key Idea: y = wᵀx + ϵ

  • wᵀx → true relationship (signal)
  • ϵ → random noise

Data Generation Model

  • Dataset: Features: xi ∈ Rd, Labels: yi ∈ R
  • Generative Equation: y = wᵀx + ϵ
  • Interpretation: Structural part → wᵀx, Noise part → ϵ

Example: House Price Prediction → Prices affected by random factors (market, demand, etc.)

Assumptions on Noise

  • Gaussian Noise: ϵ ∼ N(0, σ²)
  • Mean = 0
  • Variance = σ²
  • Distribution of y: yi ∼ N(wᵀxi, σ²)

Output is Gaussian, centered around prediction

i.i.d Assumption: Data points are independent and identically distributed

Maximum Likelihood Estimation (MLE)

  • Goal: Find w that makes observed data most probable

Likelihood Function:

L(w) = Πᵢ (1/√(2πσ²)) exp( −(yᵢ−wᵀxᵢ)²/2σ² )

Log-Likelihood:

log L(w) = Σᵢ ( −(yᵢ−wᵀxᵢ)²/2σ² − log(√(2πσ²)) )

Key Result:

  • Maximizing log-likelihood ≡ minimizing: Σ(yᵢ−wᵀxᵢ)²

Same as least squares error

Final Estimator

ŵML = (XXᵀ)†Xy

Same as standard linear regression solution

Key Insight: Loss Function = Noise Assumption

  • Choosing a loss function = assuming a noise distribution
  • Gaussian Noise → Squared Error: (y−wᵀx)²
  • Laplace Noise → Absolute Error: |y−wᵀx|

Intuition:

  • Gaussian → penalizes large errors more
  • Laplace → more robust to outliers

Why This View is Important

  • Provides theoretical justification for squared error
  • Connects ML with statistics & probability
  • Helps design better models (choose noise wisely)

Final Intuition

Linear Regression =

  • Assume data = signal + noise
  • Use probability to estimate best w
  • Minimizing squared error = most likely explanation of data
