Lecture 1: Perceptron Learning Algorithm

Supervised Learning (Binary Classification)

Generative Models

  • Learn how data is generated.
  • Model:
    • Prior: P(y)
    • Class-conditional: P(x|y)
  • Bayes theorem:

P(y|x) = P(x|y)P(y) / P(x)

  • Examples:
    • Naive Bayes (assumes feature independence)
    • Gaussian Discriminant Analysis (continuous data)

Discriminative Models

  • Learn direct mapping from input to output.
  • Directly model: P(y|x)
  • Examples:
    • K-Nearest Neighbors (KNN)
    • Decision Trees

Why Linear Discriminative Models?

  • Generative models (e.g., Naive Bayes) often lead to linear decision boundaries.
  • Instead of modeling the full distribution, we directly learn a linear function.
  • Decision boundary: wᵀx = 0

Linear Separability Assumption

Decision Rule:

y = +1 if wᵀx ≥ 0; −1 otherwise

  • Definition: Data is linearly separable if a line (or hyperplane) can perfectly separate classes.
  • Limitation: Fails when data overlaps or outliers exist.

Computational Complexity

  • General case: Minimizing classification error (0−1 loss) is NP-hard.
  • Special case (linearly separable): A perfect separator exists → problem becomes solvable efficiently.

Perceptron Algorithm

Idea

  • Inspired by biological neurons.
  • Learns a linear classifier.

Input Dataset: {(x₁,y₁), (x₂,y₂), …, (xₙ,yₙ)} where xᵢ ∈ Rᵈ, yᵢ ∈ {+1,−1}.

Initialization w = 0

Prediction ŷ = sign(wᵀx)

Update Rule

  • If correct prediction: No change.
  • If wrong prediction: wt+1 = wt + yᵢxᵢ

Convergence

  • Guaranteed only if data is linearly separable.
  • May not converge for non-separable data.
Simple Example
xy
(2,2)+1
(-1,-1)-1

Step:

  • Start: w=(0,0)
  • For (2,2), y=+1:
    • Prediction = 0 → wrong
    • Update: w = (0,0) + (1)(2,2) = (2,2)
  • Now it starts classifying correctly.

Key Takeaways

  • Generative → model data distribution.
  • Discriminative → model decision boundary.
  • Linear models assume separability.
  • Perceptron:
    • Simple & fast.
    • Works only for separable data.
    • Updates weights on mistakes.

Lecture 2: Understanding Perceptron Update Rule

Perceptron Update Rule

  • Core Idea: Perceptron updates weights only when it makes a mistake.
  • Update Formula: wt+1 = wt + yᵢxᵢ

Where:

  • wt: current weight vector
  • xᵢ: input vector
  • yᵢ ∈ {+1,−1}: true label
  • Prediction Rule: ŷ = sign(wᵀx)

Types of Mistakes

Type 1 Mistake (False Positive)

  • Predicted: +1 → wᵀxᵢ ≥ 0
  • Actual: −1
  • Update: wt+1 = wt − xᵢ
  • Effect: wt+1ᵀxᵢ = wtᵀxᵢ − ‖xᵢ‖² → Dot product decreases, pushing toward the correct class.

Type 2 Mistake (False Negative)

  • Predicted: −1 → wᵀxᵢ < 0
  • Actual: +1
  • Update: wt+1 = wt + xᵢ
  • Effect: wt+1ᵀxᵢ = wtᵀxᵢ + ‖xᵢ‖² → Dot product increases, pushing toward the correct class.

Key Insight: Each update moves weights in the correct direction. It may not fix immediately, but progress is guaranteed for that point.

Local vs Global Problem

  • Issue: Fixing one point = local improvement, but may harm others = global issue.
  • Reason: Updating w shifts the decision boundary. Previously correct points may become incorrect.
  • Result: Algorithm may keep correcting different points and enter an infinite loop.

Linear Separability is Not Always Enough

  • Edge Case: Points lying exactly on decision boundary (wᵀx=0).
  • Issue: Sign function treats wᵀx ≥ 0 ⇒ +1, creating ambiguity.
  • Counter Example: Points on boundary cause perceptron to keep switching decisions → no stabilization.

Linear Separability with γ-Margin

Condition for Convergence: yᵢ(w*ᵀxᵢ) ≥ γ

Where:

  • γ > 0: margin
  • w*: optimal weight vector
  • Meaning: All points are not just separable, but at least γ distance away from boundary.
  • Visual Idea: Clear buffer zone between classes.

Importance:

  • Removes boundary ambiguity
  • Prevents infinite cycling
  • Guarantees convergence of perceptron
Simple Example
PointLabel
(2,2)+1
(-2,-1)-1
  • If points are far from boundary → large margin (γ) → perceptron quickly converges.
  • If point lies near/on boundary → may oscillate.

Key Takeaways

  • Perceptron updates only on mistakes.
  • Two mistake types handled by same formula.
  • Updates move weights in correct direction.
  • Local fixes may break global classification.
  • Linear separability alone is insufficient.
  • γ-margin ensures stability, convergence, and no ambiguity.

Lecture 3: Proof of Convergence of Perceptron Algorithm

Core Assumptions

To simplify the proof, we assume:

  1. Linear Separability with Margin (γ): yᵢ(w*ᵀxᵢ) ≥ γ, γ > 0
    • Data is perfectly separable.
    • There is a clear gap (margin γ) from the boundary.
  2. Radius Assumption (R): ‖xᵢ‖ ≤ R
    • All points lie inside a sphere of radius R.
  3. Unit Length of Optimal Vector ‖w*‖ = 1
    • Scaling w* does not change the boundary, only rescales γ.
    • Hence, we can fix ‖w*‖ = 1.

Perceptron Update Rule

  • Initialization: w₀ = 0
  • Update (only on mistake): wt+1 = wt + yᵢxᵢ

Goal of Proof

Show that the total number of mistakes L is finite. If mistakes are bounded, the algorithm must converge.

Observation 1: Upper Bound on Weight Length

  • Track growth of: ‖wL‖²
  • Result: ‖wL‖² ≤ L·R²
  • Meaning: Weight vector cannot grow too fast. Each mistake increases length by at most R².

Observation 2: Alignment with Optimal Vector

  • Track: wLᵀw*
  • Result: wLᵀw* ≥ L·γ
  • Meaning: Each update improves alignment with the true separator. Progress is consistent and positive.

Combining Both → Radius−Margin Bound

Using Cauchy−Schwarz Inequality: (wLᵀw*)² ≤ ‖wL‖²·‖w*‖²

Since ‖w*‖ = 1: (Lγ)² ≤ ‖wL‖²

Using Observation 1: L²γ² ≤ LR²

Final Result: L ≤ R²/γ²

Interpretation

Number of mistakes depends on:

  • R (data spread)
  • γ (margin size)
CaseEffect
Large γ (big margin)Faster convergence
Small γ (tiny margin)Slow convergence
γ = 0No guarantee
Simple Example
  • Case 1: Good Margin: γ = 1, R = 2 → L ≤ 4/1 = 4. Few mistakes → fast learning.
  • Case 2: Small Margin: γ = 0.1, R = 2 → L ≤ 4/0.01 = 400. Many mistakes → slow learning.

Key Takeaways

  • Perceptron converges only if margin exists.
  • Mistakes are bounded by: R²/γ²
  • Larger margin → better learning.
  • If γ=0, convergence guarantee breaks.
  • Foundation for:
    • Neural Networks
    • Support Vector Machines (SVMs)
Final Intuition: Perceptron learns faster when classes are clearly separated with a large margin.

Lecture 4: Sigmoid Function for Modeling Class Probabilities

Limitations of Perceptron

  • Assumption: Data must be linearly separable.
  • Mistake Bound: Mistakes ≤ R²/γ²

Problem: Perceptron assumes:

P(y=1|x) = 1 if wᵀx ≥ 0; 0 if wᵀx < 0

This is too strict.

  • Cannot handle noisy data.
  • Cannot handle overlapping classes.
  • Probabilities are only 0 or 1.

Linear Score

  • Definition: Z = wᵀx
  • Interpretation:
    • Z > 0: leaning towards class +1
    • Z < 0: leaning towards class −1
    • Z = 0: decision boundary

Link Function (Sigmoid)

  • Need: Convert Z ∈ (−∞, ∞) into probability in [0,1].
  • Sigmoid Function:

G(Z) = 1 / (1 + e⁻ᶻ)

Properties:

  • G(0) = 0.5 → uncertainty
  • Z→∞ ⇒ G(Z) → 1
  • Z→−∞ ⇒ G(Z) → 0

Modeling Probabilities

Final Model:

P(y=1|x) = 1 / (1 + e⁻ʷᵀˣ)

P(y=−1|x) = 1 − P(y=1|x)

  • Key Idea: Every point has non-zero probability.
  • Interpretation: Like a biased coin, where bias depends on wᵀx.
Examples
Example 1: wᵀx = 1 → P(y=1) = 1/(1+e⁻¹) ≈ 0.73
Example 2: wᵀx = 10 → P(y=1) ≈ 0.99995

Insight:

  • Farther from boundary → higher confidence.
  • Closer to boundary → more uncertainty.

Maximum Likelihood Estimation (MLE)

  • Goal: Find best w such that likelihood is maximized.
  • Likelihood Function:

L(w) = Π P(yᵢ|xᵢ), for i = 1 to n

Log-Likelihood (used in practice):

ℓ(w) = Σ log P(yᵢ|xᵢ), for i = 1 to n

Objective:

max over w of ℓ(w)

Key Takeaways

  • Perceptron → hard classification (0 or 1).
  • Logistic Regression → probabilistic model.
  • Uses:
    • Linear score wᵀx.
    • Sigmoid function.
  • Works well for:
    • Noisy data.
    • Non-separable data.
Final Intuition: Logistic regression replaces hard decisions with smooth probabilities.

Lecture 5: Sigmoid Function for Modeling Class Probabilities (continued)

Core Concepts

Sigmoid (Logistic) Function: Converts any real value into a probability in [0,1]:

σ(z) = 1 / (1 + e⁻ᶻ)

Probabilistic Model

P(y=1|x) = 1 / (1 + e⁻ʷᵀˣ)

  • Output is a probability (confidence).
  • Unlike perceptron, not restricted to 0 or 1.

Maximum Likelihood Estimation (MLE)

  • Goal: Find w that makes observed labels most likely.

max over w of P(data|w)

Gradient Intuition

  • Gradient is a weighted sum of data points.
  • Points with large error → stronger updates.

Confidence Interpretation

  • Output near 1 → high confidence.
  • Output near 0.5 → uncertain.

Key Formulae

  1. Probability Model

    P(y=1|x) = 1 / (1 + e⁻ʷᵀˣ)

  2. Log-Likelihood Function

    ℓ(w) = Σ [(1−yᵢ)(−wᵀxᵢ) − log(1+e⁻ʷᵀˣⁱ)], for i = 1 to n

  3. Gradient of Log-Likelihood

    ∇ℓ(w) = Σ xᵢ(yᵢ − 1/(1+e⁻ʷᵀˣⁱ)), for i = 1 to n

  4. Gradient Ascent Update

    wt+1 = wt + ηₜ∇ℓ(wt)

    where ηₜ is the learning rate.

  5. Regularized Logistic Regression

    min over w of Σ[log(1+e⁻ʷᵀˣⁱ) + (1−yᵢ)wᵀxᵢ] + λ‖w‖², for i = 1 to n

    • λ controls overfitting.

Examples

Prediction Example
For test point xtest:
  1. Compute score: z = wᵀxtest
  2. Probability: P(y=1) = σ(z)
  3. Final prediction: ŷ = sign(wᵀxtest)
Mismatch Example
  • True label: y=1
  • Predicted: P(y=1)=0.1
  • Error: 1−0.1=0.9
  • This point strongly influences the update.
Insight: Bigger mistake → bigger correction.

Handling Non-Linearity

  • Logistic regression can be kernelized.
  • Represent: w = Σ αᵢxᵢ
  • Allows non-linear decision boundaries and complex patterns.

Key Takeaways

  • Logistic regression is a probabilistic classifier.
  • Uses:
    • Linear score wᵀx.
    • Sigmoid function.
  • Trained using:
    • MLE + Gradient ascent.
  • Handles:
    • Noise.
    • Overlapping data.
  • Regularization prevents overfitting.
Final Intuition: Logistic regression predicts how likely a class is, not just which class.
